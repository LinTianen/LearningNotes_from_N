# Markdown 基本语法完整指南
你需要掌握Markdown的核心基础语法，以下内容涵盖日常编写最常用的格式，清晰易懂且可直接套用。

## 一、标题
用于划分文档层级，支持1-6级标题，通过`#`数量区分层级，`#`后需加一个空格。
```markdown
# 1级标题（最大号，对应HTML <h1>）
## 2级标题
### 3级标题
#### 4级标题
##### 5级标题
###### 6级标题（最小号，对应HTML <h6>）
```
> 注意：实际使用中，建议1级标题只使用1次，避免层级混乱。

## 二、文本格式
### 1. 加粗
用于强调重要内容，两种实现方式：
```markdown
**这是加粗文本**（推荐，兼容性更好）
__这也是加粗文本__
```
效果：**这是加粗文本**

### 2. 斜体
用于轻微强调或标注，两种实现方式：
```markdown
*这是斜体文本*（推荐）
_这也是斜体文本_
```
效果：*这是斜体文本*

### 3. 加粗+斜体
组合使用，强化强调效果：
```markdown
***这是加粗+斜体文本***
___这也是加粗+斜体文本___
**_这同样是加粗+斜体文本_**
```
效果：***这是加粗+斜体文本***

### 4. 删除线
用于标记作废、已删除的内容：
```markdown
~~这是带删除线的文本~~
```
效果：~~这是带删除线的文本~~
### 5. 下划线（基于html）
```markdown
<u>下划线文本</u>
```
效果：<u>这是下划线文本</u>

## 三、列表
### 1. 无序列表
用于无需排序的内容集合，支持`-`、`*`、`+`三种标记，标记后需加空格。
```markdown
- 列表项1
- 列表项2
  - 子列表项（缩进2个空格或1个Tab）
  - 子列表项
* 列表项3
+ 列表项4
```
效果：
- 列表项1
- 列表项2
  - 子列表项
  - 子列表项
* 列表项3
+ 列表项4

### 2. 有序列表
用于需要排序的内容，以「数字+`.`+空格」开头，数字顺序不影响渲染结果。
```markdown
1. 第一步
2. 第二步
   1. 子步骤1
   2. 子步骤2
3. 第三步（即使写成5.，渲染仍为3.）
```
效果：
1. 第一步
2. 第二步
   1. 子步骤1
   2. 子步骤2
3. 第三步

### 3. 任务列表（Task List）
用于标记待办事项完成状态，`-`后加`[ ]`（未完成）或`[x]`（已完成），中间需无空格。
```markdown
- [ ] 未完成任务1
- [ ] 未完成任务2
- [x] 已完成任务1
```
效果：
- [ ] 未完成任务1
- [ ] 未完成任务2
- [x] 已完成任务1

## 四、链接（Links）
用于跳转至其他页面、文档或网址，分两种形式：

### 1. 行内链接（Inline Link）
最常用，直接嵌入链接地址：
```markdown
[链接文本](https://www.example.com "可选的标题提示（鼠标悬浮显示）")
例如：[百度](https://www.baidu.com "全球最大中文搜索引擎")
```
效果：[百度](https://www.baidu.com "全球最大中文搜索引擎")

### 2. 参考式链接（Reference Link）
适合多次引用同一链接，便于统一管理：
```markdown
// 正文部分
[链接文本][参考标识]
[另一个链接][1]

// 参考部分（可放在文档任意位置，通常在末尾）
[参考标识]: https://www.example1.com "可选标题"
[1]: https://www.example2.com
```
效果：
[链接文本][参考标识]
[另一个链接][1]

[参考标识]: https://www.example1.com "可选标题"
[1]: https://www.example2.com

## 五、图片（Images）
语法与链接类似，仅在开头多一个`!`，支持行内式和参考式。

### 1. 行内图片
```markdown
![图片加载失败时的替代文本](https://www.example.com/image.jpg "可选的图片标题")
// 本地图片：![替代文本](./images/photo.png)（路径为相对路径或绝对路径）
```
效果：![示例图片](https://picsum.photos/200/100 "示例图片")


### 2. 参考式图片
```markdown
![图片替代文本][img1]

[img1]: https://www.example.com/image.jpg "图片标题"
```

## 六、代码（Code）
### 1. 行内代码（Inline Code）
用于标记单行代码、命令或变量，用反引号`` ` ``包裹：
```markdown
在终端输入 `npm install` 安装依赖，变量 `name` 用于存储用户名。
```
效果：在终端输入 `npm install` 安装依赖，变量 `name` 用于存储用户名。

### 2. 代码块（Code Block）
用于展示多行代码，支持语法高亮，有两种实现方式：

#### 方式1：三个反引号（推荐，支持指定语言）
```markdown
```javascript
// JavaScript 代码示例
function sayHello() {
  console.log("Hello, Markdown!");
}
sayHello();
```

```python
# Python 代码示例
def say_hello():
    print("Hello, Markdown!")
say_hello()
```
```
效果：
```javascript
// JavaScript 代码示例
function sayHello() {
  console.log("Hello, Markdown!");
}
sayHello();
```
```

```python
# Python 代码示例
def say_hello():
    print("Hello, Markdown!")
say_hello()
```
```

#### 方式2：缩进式（每行缩进4个空格或1个Tab）
```markdown
    # Python 代码示例
    def say_hello():
        print("Hello, Markdown!")
    say_hello()
```
> 注意：缩进式不支持语法高亮，推荐使用反引号方式。

## 七、引用（Blockquotes）
用于引用他人观点、文献内容，用`>`开头，支持多层嵌套。
```markdown
> 这是一级引用（对应HTML <blockquote>）
> > 这是二级引用（嵌套引用，缩进效果）
> > > 这是三级引用
> 
> 引用内容可以换行，只需在换行后继续加>，或直接换行（空行需补>）
```
效果：
> 这是一级引用
> > 这是二级引用
> > > 这是三级引用
> 
> 引用内容可以换行，只需在换行后继续加>，或直接换行（空行需补>）

## 八、分割线（Horizontal Rules）
用于分隔文档内容，划分不同模块，三种实现方式效果一致：
```markdown
---（推荐，3个及以上短横线）
***（3个及以上星号）
___（3个及以上下划线）
```
效果：
***
## 九、表格（Tables）
用于展示结构化数据，`|`分隔列，`-`分隔表头和内容，支持对齐方式设置。
```markdown
| 姓名   | 年龄 | 职业       |
| ------ | ---- | ---------- |
| 张三   | 25   | 程序员     |
| 李四   | 30   | 设计师     |
| 王五   | 28   | 产品经理   |

// 自定义对齐方式（: 表示对齐方向）
| 左对齐 | 居中对齐 | 右对齐 |
| :----- | :------: | -----: |
| 内容1  | 内容2    | 内容3  |
| 内容4  | 内容5    | 内容6  |
```
效果：
| 姓名   | 年龄 | 职业       |
| ------ | ---- | ---------- |
| 张三   | 25   | 程序员     |
| 李四   | 30   | 设计师     |
| 王五   | 28   | 产品经理   |

| 左对齐 | 居中对齐 | 右对齐 |
| :----- | :------: | -----: |
| 内容1  | 内容2    | 内容3  |
| 内容4  | 内容5    | 内容6  |

## 十、转义字符（Escaping Characters）
当需要显示Markdown语法符号本身（如`#`、`*`、`[`等）时，在符号前加反斜杠`\`转义：
```markdown
\# 这不是标题（显示#本身）
\* 这不是斜体（显示*本身）
\[链接文本\](https://www.example.com)（显示[]本身，不渲染为链接）
```
效果：
\# 这不是标题
\* 这不是斜体
\[链接文本\](https://www.example.com)

### 总结
1. Markdown语法简洁直观，核心包含标题、文本样式、列表、链接、图片、代码等10大类常用格式；
2. 语法兼容性强，大部分编辑器（VS Code、Typora等）均支持，无需复杂配置即可使用；
3. 灵活组合各类语法可满足文档编写、笔记记录、博客创作等大部分场景需求。
