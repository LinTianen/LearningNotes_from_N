# Markdown 基本语法完整指南
你需要掌握Markdown的核心基础语法，以下内容涵盖日常编写最常用的格式，清晰易懂且可直接套用。

## 一、标题
用于划分文档层级，支持1-6级标题，通过`#`数量区分层级，`#`后需加一个空格。
```markdown
# 1级标题（最大号，对应HTML <h1>）
## 2级标题
### 3级标题
#### 4级标题
##### 5级标题
###### 6级标题（最小号，对应HTML <h6>）
```
> 注意：实际使用中，建议1级标题只使用1次，避免层级混乱。

## 二、文本格式
### 1. 加粗
用于强调重要内容，两种实现方式：
```markdown
**这是加粗文本**（推荐，兼容性更好）
__这也是加粗文本__
```
效果：**这是加粗文本**

### 2. 斜体
用于轻微强调或标注，两种实现方式：
```markdown
*这是斜体文本*（推荐）
_这也是斜体文本_
```
效果：*这是斜体文本*

### 3. 加粗+斜体
组合使用，强化强调效果：
```markdown
***这是加粗+斜体文本***
___这也是加粗+斜体文本___
**_这同样是加粗+斜体文本_**
```
效果：***这是加粗+斜体文本***

### 4. 删除线
用于标记作废、已删除的内容：
```markdown
~~这是带删除线的文本~~
```
效果：~~这是带删除线的文本~~
### 5. 下划线（基于html）
```markdown
<u>下划线文本</u>
```
效果：<u>这是下划线文本</u>

## 三、列表
### 1. 无序列表
用于无需排序的内容集合，支持`-`、`*`、`+`三种标记，标记后需加空格。
```markdown
- 列表项1
- 列表项2
  - 子列表项（缩进2个空格或1个Tab）
  - 子列表项
* 列表项3
+ 列表项4
```
效果：
- 列表项1
- 列表项2
  - 子列表项
  - 子列表项
* 列表项3
+ 列表项4

### 2. 有序列表
用于需要排序的内容，以「数字+`.`+空格」开头，数字顺序不影响渲染结果。
```markdown
1. 第一步
2. 第二步
   1. 子步骤1
   2. 子步骤2
3. 第三步（即使写成5.，渲染仍为3.）
```
效果：
1. 第一步
2. 第二步
   1. 子步骤1
   2. 子步骤2
3. 第三步

### 3. 任务列表（Task List）
用于标记待办事项完成状态，`-`后加`[ ]`（未完成）或`[x]`（已完成），中间需无空格。
```markdown
- [ ] 未完成任务1
- [ ] 未完成任务2
- [x] 已完成任务1
```
效果：
- [ ] 未完成任务1
- [ ] 未完成任务2
- [x] 已完成任务1

## 四、链接（Links）
用于跳转至其他页面、文档或网址，分两种形式：

### 1. 行内链接（Inline Link）
最常用，直接嵌入链接地址：
```markdown
[链接文本](https://www.example.com "可选的标题提示（鼠标悬浮显示）")
例如：[百度](https://www.baidu.com "全球最大中文搜索引擎")
```
效果：[百度](https://www.baidu.com "全球最大中文搜索引擎")

### 2. 参考式链接（Reference Link）
适合多次引用同一链接，便于统一管理：
```markdown
// 正文部分
[链接文本][参考标识]
[另一个链接][1]

// 参考部分（可放在文档任意位置，通常在末尾）
[参考标识]: https://www.example1.com "可选标题"
[1]: https://www.example2.com
```
效果：
[链接文本][参考标识]
[另一个链接][1]

[参考标识]: https://www.example1.com "可选标题"
[1]: https://www.example2.com

## 五、图片（Images）
语法与链接类似，仅在开头多一个`!`，支持行内式和参考式。

### 1. 行内图片
```markdown
![图片加载失败时的替代文本](https://www.example.com/image.jpg "可选的图片标题")
// 本地图片：![替代文本](./images/photo.png)（路径为相对路径或绝对路径）
```
效果：![示例图片](https://picsum.photos/200/100 "示例图片")


### 2. 参考式图片
```markdown
![图片替代文本][img1]

[img1]: https://www.example.com/image.jpg "图片标题"
```

## 六、代码（Code）
### 1. 行内代码（Inline Code）
用于标记单行代码、命令或变量，用反引号`` ` ``包裹：
```markdown
在终端输入 `npm install` 安装依赖，变量 `name` 用于存储用户名。
```
效果：在终端输入 `npm install` 安装依赖，变量 `name` 用于存储用户名。

### 2. 代码块（Code Block）
用于展示多行代码，支持语法高亮，有两种实现方式：

#### 方式1：三个反引号（推荐，支持指定语言）
```markdown
```javascript
// JavaScript 代码示例
function sayHello() {
  console.log("Hello, Markdown!");
}
sayHello();
```

```python
# Python 代码示例
def say_hello():
    print("Hello, Markdown!")
say_hello()
```
```
效果：
```javascript
// JavaScript 代码示例
function sayHello() {
  console.log("Hello, Markdown!");
}
sayHello();
```
```

```python
# Python 代码示例
def say_hello():
    print("Hello, Markdown!")
say_hello()
```
```

#### 方式2：缩进式（每行缩进4个空格或1个Tab）
```markdown
    # Python 代码示例
    def say_hello():
        print("Hello, Markdown!")
    say_hello()
```
> 注意：缩进式不支持语法高亮，推荐使用反引号方式。

## 七、引用（Blockquotes）
用于引用他人观点、文献内容，用`>`开头，支持多层嵌套。
```markdown
> 这是一级引用（对应HTML <blockquote>）
> > 这是二级引用（嵌套引用，缩进效果）
> > > 这是三级引用
> 
> 引用内容可以换行，只需在换行后继续加>，或直接换行（空行需补>）
```
效果：
> 这是一级引用
> > 这是二级引用
> > > 这是三级引用
> 
> 引用内容可以换行，只需在换行后继续加>，或直接换行（空行需补>）

## 八、分割线（Horizontal Rules）
用于分隔文档内容，划分不同模块，三种实现方式效果一致：
```markdown
---（推荐，3个及以上短横线）
***（3个及以上星号）
___（3个及以上下划线）
```
效果：
***
## 九、表格（Tables）
用于展示结构化数据，`|`分隔列，`-`分隔表头和内容，支持对齐方式设置。
```markdown
| 姓名   | 年龄 | 职业       |
| ------ | ---- | ---------- |
| 张三   | 25   | 程序员     |
| 李四   | 30   | 设计师     |
| 王五   | 28   | 产品经理   |

// 自定义对齐方式（: 表示对齐方向）
| 左对齐 | 居中对齐 | 右对齐 |
| :----- | :------: | -----: |
| 内容1  | 内容2    | 内容3  |
| 内容4  | 内容5    | 内容6  |
```
效果：
| 姓名   | 年龄 | 职业       |
| ------ | ---- | ---------- |
| 张三   | 25   | 程序员     |
| 李四   | 30   | 设计师     |
| 王五   | 28   | 产品经理   |

| 左对齐 | 居中对齐 | 右对齐 |
| :----- | :------: | -----: |
| 内容1  | 内容2    | 内容3  |
| 内容4  | 内容5    | 内容6  |

## 十、转义字符（Escaping Characters）
当需要显示Markdown语法符号本身（如`#`、`*`、`[`等）时，在符号前加反斜杠`\`转义：
```markdown
\# 这不是标题（显示#本身）
\* 这不是斜体（显示*本身）
\[链接文本\](https://www.example.com)（显示[]本身，不渲染为链接）
```
效果：
\# 这不是标题
\* 这不是斜体
\[链接文本\](https://www.example.com)

### 总结
1. Markdown语法简洁直观，核心包含标题、文本样式、列表、链接、图片、代码等10大类常用格式；
2. 语法兼容性强，大部分编辑器（VS Code、Typora等）均支持，无需复杂配置即可使用；
3. 灵活组合各类语法可满足文档编写、笔记记录、博客创作等大部分场景需求。
  
# VSCode Markdown 彩色字体完整方案
标准 Markdown 无原生改色语法，VSCode 预览支持 **HTML 内嵌样式（通用无插件）**、**专用扩展（极简语法）**、**全局自定义预览CSS** 三种方案，下面按推荐度排序。

## 一、通用方案：HTML `<span>`（无需插件，所有VSCode版本生效）
VSCode 默认预览完全支持内联 HTML，兼容性最强，导出HTML/PDF也保留颜色。
### 基础语法
```markdown
<!-- 英文颜色名 -->
<span style="color:red">红色文字</span>
<span style="color:green">绿色文字</span>
<span style="color:blue">蓝色文字</span>

<!-- 十六进制色（推荐，自定义精准颜色） -->
<span style="color:#f44336">砖红</span>
<span style="color:#2196F3">天蓝</span>
<span style="color:#4CAF50">翠绿</span>

<!-- RGB / RGBA 带透明度 -->
<span style="color:rgb(255, 153, 0)">橙色</span>
<span style="color:rgba(100, 180, 255, 0.8)">半透浅蓝</span>

<!-- 同时设置字体颜色+背景高亮+加粗 -->
<span style="color:white;background:#d32f2f;font-weight:bold">警告高亮文字</span>
```
### 效果说明
1. 打开预览：`Ctrl+Shift+V` / `Ctrl+K V` 分栏预览，立刻渲染彩色；
2. 缺点：代码较长，写起来繁琐；
3. 注意：`<font color="xxx">` 是HTML废弃标签，不推荐使用。

## 二、极简方案：安装扩展（短语法，写文档更舒服）
### 1. Markdown MojiColor（最热门）
市场搜索 `Markdown MojiColor` 安装，语法超简短：
```markdown
%红色文字%{red}
%自定义蓝色%{#0099ff}
%半透紫%{rgba(150,50,200,0.7)}
```
无需写长span标签，预览自动解析，仅VSCode本地生效。

### 2. markdown-helper 快捷插入
安装后选中文字，快捷键 `Ctrl+Shift+C`，一键弹出选色自动包裹颜色标签。

## 三、全局统一样式：自定义预览CSS（批量改标题/代码块颜色）
如果想让**所有一级标题统一蓝色、引用块灰色**，不用每次写span：
1. 项目根目录新建 `md-custom.css`
```css
/* 自定义所有h1标题颜色 */
h1 { color: #007acc; }
/* 引用块文字颜色 */
blockquote { color: #666; border-left:4px #007acc solid; }
/* 行内代码颜色 */
code { color:#e91e63; }
```
2. 打开VSCode设置 `settings.json`（`Ctrl+,` 打开设置，右上角打开json），添加：
```json
"markdown.styles": ["./md-custom.css"]
```
保存后重启预览，全局Markdown元素统一变色。

## 四、常见问题
1. **预览没颜色？**
   - 检查HTML标签引号是否成对，style语法无拼写错误；
   - 关闭再重新打开预览窗口刷新；
2. **GitHub/Gitee上传后颜色消失？**
   GitHub 安全策略屏蔽内联style，仅本地VSCode预览生效；若要网页展示彩色只能用图片。
3. **代码块内无法上色？**
   ``` ```代码块``` ``` 内部是纯文本，HTML标签不会解析；如需代码内标红要用 `<pre>` 包裹纯HTML段落。

# 超链接
## 一、原生 Markdown 超链接语法（通用，所有编辑器都生效）
### 1. 基础行内链接（最常用：网页/外部地址）
**语法格式**
```markdown
[链接显示文字](目标URL "鼠标悬停提示文字")
```
- 括号内 `"提示文字"` 为可选，可省略；
- 英文括号、英文引号，不要用中文符号。

**示例**
```markdown
[百度](https://www.baidu.com)
[哔哩哔哩](https://www.bilibili.com "B站官网")
```

### 2. 纯网址自动链接
直接粘贴完整 URL，多数 Markdown 渲染器会自动转为可点击链接：
```markdown
https://www.baidu.com
```

### 3. 链接到**本地文件/本地路径**
常用于链接同目录、子目录的文档、图片、文件，推荐使用**相对路径**（兼容性最好）。
```markdown
# 同目录文件
[本地文档](./笔记.md)

# 子目录文件
[图片](./images/demo.png)

# 路径含空格时，用 <> 包裹地址
[带空格的文件](<./我的 文档.txt>)
```

### 4. 文档内锚点（页内跳转，跳转到当前文档标题）
可以点击链接跳转到本文档的指定标题位置，VS Code 预览也支持。
规则：标题转锚点时，**英文小写、空格变 `-`、特殊符号剔除**。

**示例**
```markdown
# 目录
[跳转到基础语法](#基础语法)

## 基础语法
这里是目标位置
```

### 5. 邮箱链接
使用 `mailto:` 协议，点击唤起邮件客户端：
```markdown
[联系我](mailto:xxx@163.com)
```

### 6. 引用式链接（长链接复用，让文档更整洁）
适合同一个链接多次使用的场景，链接地址统一写在文末：
```markdown
[掘金][juejin]
[我的主页][juejin]

# 文末定义链接
[juejin]: https://juejin.cn
```

## 二、VS Code 自带快捷插入（不用手写语法）
不用记语法，图形化操作更省事：
### 方式1：右键菜单
1. 选中要作为**链接文字**的内容；
2. 右键 → 选择 **插入链接(Insert Link)**；
3. 在弹出输入框中粘贴/填写目标地址，回车即可自动生成标准链接语法。

### 方式2：系统快捷键（推荐）
先选中文字，再按下组合键：
- Windows / Linux：`Ctrl + K` 松开，再按 `Ctrl + L`
- Mac：`Cmd + K` 松开，再按 `Cmd + L`

之后输入链接地址，快速完成插入。

### 查看链接效果
打开 Markdown 预览面板：
快捷键 `Ctrl + Shift + V`（Win/Linux）/ `Cmd + Shift + V`（Mac），右侧会实时渲染，**点击链接即可正常跳转**（网页调用浏览器、本地文件调用默认程序）。

## 常见小问题排查
1. **链接点了没反应**
   - 检查是否用了**中文括号/中文引号**，必须使用英文符号；
   - 本地路径写错（区分大小写、层级），建议优先用 `./` 相对路径。
2. **路径有空格失效**
   用 `<地址>` 包裹路径：`[文件](<./名字 带空格.md>)`。
3. **锚点跳转失败**
   标题不要加特殊符号，锚点名称和标题严格对应（空格转 `-`、全小写）。
