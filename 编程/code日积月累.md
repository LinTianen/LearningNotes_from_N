
在 C++ 里，同一个类里重载两次 `operator[]`，分别是**非常量版本**和**常量版本**，是为了让 `[]` 运算符在**普通对象**和**const对象**上都能正确、安全地使用。


### 两个版本的区别与作用

```cpp
// 1. 非常量版本
double& operator[](int nIndex);

// 2. 常量版本
const double& operator[](int nIndex) const;
```

| 版本 | 调用时机 | 返回值 | 作用 |
| :--- | :--- | :--- | :--- |
| `double& operator[]` | 当对象是**非const**时调用 | 非常量引用 | 可读、可写（可修改元素） |
| `const double& operator[] const` | 当对象是**const**时调用 | 常量引用 | 只能读，不能修改元素 |

### 为什么要这么设计？

1.  **支持const对象安全访问**
    如果没有 `const` 版本，下面这种代码会编译报错：
    ```cpp
    const DArray arr(10); // const对象
    double val = arr[0];  // 调用const版本，只能读
    arr[0] = 3.14;        // 非法，const版本返回const引用，不能赋值
    ```
    这样能保证const对象的状态不会被意外修改。

2.  **支持非const对象的读写**
    对于普通对象，会优先调用非常量版本：
    ```cpp
    DArray arr(10);
    arr[0] = 3.14; // 调用非常量版本，返回引用，可以修改元素
    double val = arr[0]; // 同样调用非常量版本，可读
    ```

3.  **实现“const正确性”**
    这是C++中一个很重要的设计原则：
    -   const对象只能调用const成员函数，也只能通过const成员函数访问数据。
    -   非const对象可以调用const和非const成员函数，优先选择非const版本。
    两个版本的 `operator[]` 配合，让你的类既可以被普通对象修改，也可以被const对象安全读取。


### 补充说明
-   两个版本的实现逻辑基本一样，都是根据索引返回对应元素的引用。
-   它们构成了**重载决议**：编译器会根据调用时对象的const属性，自动选择调用哪一个版本。
