# 第8章 类属机制——模板
## 一、类属（泛型）程序设计基础
### 1. 需求背景
程序中常存在**功能逻辑完全相同、仅操作数据类型不同**的程序实体（函数、类），重复编写会造成代码冗余，例如：
- 不同类型数组的排序函数：`int_sort`、`double_sort`
- 不同元素类型的栈类：`IntStack`、`DoubleStack`

### 2. 核心概念
- **类属性（generics）**：一个程序实体能对多种类型的数据进行操作或描述的特性。
- **类属程序设计（泛型程序设计）**：基于具有类属性的程序实体进行程序设计的技术。
- 类属实体分类：**类属函数**、**类属类**。

### 3. C++类属函数的实现机制
1.  通用指针类型参数 + 函数指针
2.  **函数模板**（核心主流机制）


## 二、函数模板
### 1. 基本定义语法
函数模板是**带有类型参数**的函数定义，一套代码适配多种数据类型。
```cpp
template <class T1, class T2, ...> 
<返回值类型> <函数名>(<参数表>)
{ 
    // 函数体：T1、T2可用于参数类型、返回值类型、局部变量类型
}
```
**语法要点**：
- `template` 是模板定义的关键字
- `< >` 内为模板参数列表，`class` 用于声明类型参数（也可替换为`typename`，效果一致）
- T1、T2 为**类型参数**，代表任意数据类型，在函数内可像普通类型一样使用

**示例：通用排序函数模板**
```cpp
template <class T> 
void sort(T elements[], unsigned int count) {
    // 元素交换
    T temp = elements[i];
    elements[i] = elements[j];
    elements[j] = temp;
}
```

### 2. 函数模板的实例化
函数模板本身不是可执行函数，使用时需要**实例化**生成对应具体类型的函数（称为**模板函数**），实例化分两种方式：

#### （1）隐式实例化
编译器根据调用时的实参类型，自动推导类型参数并生成对应模板函数。
```cpp
int a[100];
sort(a, 100);  
// 自动生成 void sort(int elements[], unsigned int count)
double b[200];
sort(b, 200);  
// 自动生成 void sort(double elements[], unsigned int count)
A c[300];
sort(c, 300);  
//生成A类型版本；要求A重载了<运算符，必要时需自定义拷贝构造和赋值运算符
```

#### （2）显式实例化🥦
当编译器无法通过实参推导类型时，手动指定类型参数完成实例化。
**语法**：`函数名<类型实参>(实参表)`
```cpp
int a[100];
sort<int>(a, 100);     // 显式指定T为int
sort<double>(b, 200);  // 显式指定T为double
```

### 3. 类型参数匹配问题与解决方案
**问题场景**：单类型参数的函数模板，传入两个不同类型实参时，编译器无法推导统一类型，出现二义性。
```cpp
template <class T> 
T max (T a, T b) { return a > b ? a : b; }

int x; double m;
max(x, m);  // 编译错误：无法确定T是int还是double
```

**三种解决方法**：
1.  **显式实例化**：手动指定类型参数
    ```cpp
    max<double>(x, m);  // T指定为double，x自动转为double
    ```
2.  **显式类型转换**：将实参统一为同一类型
    ```cpp
    max((double)x, m);
    ```
3.  **重载普通函数**：定义对应参数的非模板重载函数
    ```cpp
    double max (int a, double b) { return a>b?a:b; }
    ```

### 4. 带非类型参数的函数模板
模板参数列表除了类型参数，还可包含**非类型参数**（编译期常量，通常为整型）。
```cpp
template <class T, int size>  // size是int型非类型参数
void f(T a) {
    T temp[size];  // 用非类型参数定义数组大小
}

// 调用时必须显式指定所有模板参数
f<int, 10>(1);  // T=int，size=10
```

## 三、类模板
### 1. 基本定义语法
当类的操作逻辑不变、仅成员数据类型可变时，通过类模板实现类属类。
```cpp
template <class T1, class T2, ...> 
class <类名> {
    // 类成员：T1、T2可用于数据成员、成员函数的参数/返回值类型
};
```

**示例：栈类模板**
```cpp
template <class T> 
class Stack
{
    T buffer[100];  // 元素类型由T决定
    int top;
public:
    Stack() { top = -1; }
    bool push(const T &x);
    bool pop(T &x);
};
```

### 2. 类外定义成员函数的语法
类模板的成员函数在类外实现时，必须再次声明模板参数，且类名后需携带模板参数。
```cpp
template <class T>
返回值类型 类名<T>::成员函数名(参数表)
{
    // 函数体
}
```

**示例：Stack类成员函数类外实现**
```cpp
template <class T>
bool Stack<T>::push(const T &x) {
    // 入栈逻辑
}

template <class T>
bool Stack<T>::pop(T &x) {
    // 出栈逻辑
}
```

### 3. 类模板的实例化⭐❗
类模板必须**显式指定类型参数**完成实例化，生成模板类后才能创建对象。
**语法**：`类名<类型实参> 对象名;`
```cpp
Stack<int> st1;       // 实例化int型栈类，创建对象st1
int x;
st1.push(10);
st1.pop(x);

Stack<double> st2;    // 实例化double型栈类
Stack<A> st3;         // 实例化自定义类型A的栈类
```

### 4. 类模板的静态成员特性
- 静态成员**仅属于实例化后的模板类**，不同类型的模板类之间不共享静态成员。🥦
- 每个类型实例都拥有独立的静态成员副本。

```cpp
template <class T>
class A
{
    static int x;
    T y;
};
template <class T> int A<T>::x = 0;  // 静态成员初始化

A<int> a1, a2;     // a1、a2共享同一个x
A<double> a3, a4;  // a3、a4共享另一个独立的x
```

### 5. 带非类型参数的类模板
类模板可包含非类型参数，常用于指定容器容量等编译期常量。
```cpp
template <class T, int size> 
class Stack
{
    T buffer[size];  // 非类型参数指定数组大小
    int top;
public:
    Stack() { top = -1; }
    bool push(const T &x);
    bool pop(T &x);
};

// 类外实现
template <class T, int size>
bool Stack<T, size>::push(const T &x) { ... }

// 实例化
Stack<int, 100> st1;  // 容量100的int栈
Stack<int, 200> st2;  // 容量200的int栈，与st1是不同的类
```


## 四、模板的复用与编译特性
### 1. 本质：参数化多态
模板属于**参数化多态**（区别于函数重载、虚函数的多态）：
- 模板是携带类型参数的代码骨架，传入不同类型可生成多份具体代码
- 实现“一段代码，多种类型复用”

### 2. 实例化的编译特性
- 实例化在**编译阶段**完成，编译时必须能看到模板的完整源代码
- 函数模板实例化：支持隐式、显式两种方式
- 类模板实例化：必须显式指定类型参数

### 3. 分离编译问题与解决方案
#### 问题描述
C++的`.cpp`文件独立编译。若模板声明放在头文件、实现放在`.cpp`文件，使用方仅包含头文件时，编译器看不到实现代码，无法完成实例化，会出现链接错误。

#### 标准解决方案
将**模板的声明和实现全部放在头文件**中，使用方直接包含该头文件，编译器即可在编译时看到完整代码并完成实例化。

## 五、C++标准模板库（STL）简介⭐
STL（Standard Template Library）是C++标准库提供的泛型组件集合，核心分为三大模块：**容器**、**迭代器**、**算法**。

### 1. 容器（Container）
容器是存储同类型元素的可变长度序列，由类模板实现，模板参数为元素类型。

#### （1）主要容器分类与特性
| 容器模板 | 头文件 | 底层实现 | 核心适用场景 |
|---------|--------|----------|--------------|
| `vector<T>` | `<vector>` | 动态数组 | 随机访问任意位置元素，主要在尾部增删 |
| `list<T>` | `<list>` | 双向链表 | 频繁在任意位置插入/删除元素 |
| `deque<T>` | `<deque>` | 分段连续空间 | 两端高频增删，同时需要随机访问 |
| `stack<T>` | `<stack>` | 默认基于deque | 仅尾部增删（后进先出） |
| `queue<T>` | `<queue>` | 默认基于deque | 尾增头删（先进先出） |
| `priority_queue<T>` | `<queue>` | vector + 堆 | 每次取出优先级最高（最大）的元素 |
| `map<K, V>` | `<map>` | 二叉树（红黑树） | 键值对存储，关键字唯一，按关键字排序查找 |
| `multimap<K, V>` | `<map>` | 二叉树 | 关键字可重复的键值对容器 |
| `set<T>` | `<set>` | 二叉树 | 元素即关键字，值唯一，自动排序 |
| `multiset<T>` | `<set>` | 二叉树 | 元素可重复的集合 |
| `basic_string<charT>` | `<string>` | 类vector结构 | 字符序列；`string`是char实例，`wstring`是wchar_t实例 |

#### （2）容器通用成员操作
##### 首尾元素访问
- `T& front()`：返回第一个元素的引用，适用于vector、list、deque、queue
- `T& back()`：返回最后一个元素的引用，适用同上

##### 增删操作
- 头部操作：`push_front(x)` 头部插入、`pop_front()` 头部删除，适用于list、deque
- 尾部操作：`push_back(x)` 尾部插入、`pop_back()` 尾部删除，适用于vector、list、deque

##### 迭代器获取
- `iterator begin()`：返回指向第一个元素的迭代器
- `iterator end()`：返回指向最后一个元素下一个位置的迭代器
- 适用：除queue、stack外的所有容器

##### 插入与删除
- `insert(pos, x)`：在迭代器pos位置插入单个元素
- `erase(pos)` / `erase(first, last)`：删除单个/区间内元素
- 适用：vector、list、deque、map/multimap、set/multiset、basic_string

##### 元素访问
- 下标访问 `operator[](pos)`：按索引访问，适用于vector、deque、basic_string
- 键值访问 `operator[](key)`：按关键字访问值，适用于map
- `at(pos)`：带越界检查的下标访问，适用同上
- `find(key)`：按关键字查找，返回迭代器，适用于map/multimap、set/multiset

#### （3）自定义类型作为元素的要求
若容器元素为自定义类，通常需要实现：
1.  **拷贝构造函数**和**赋值运算符重载**：容器操作会涉及元素拷贝与赋值
2.  重载**小于运算符`<`**：支持容器排序、有序容器的默认比较逻辑

### 2. 迭代器（Iterator）
迭代器是抽象的“智能指针”，指向容器元素，是连接容器与算法的桥梁，用于遍历和访问容器元素。

#### （1）迭代器分类（功能由弱到强）
1.  **输出迭代器（OutIt）**：仅支持修改元素，操作：`*it = 值`、`++`
2.  **输入迭代器（InIt）**：仅支持读取元素，操作：`值 = *it`、`->`、`++`、`==`、`!=`
3.  **前向迭代器（FwdIt）**：可读可写，仅支持向前遍历，操作：`*`、`->`、`++`、`==`、`!=`
4.  **双向迭代器（BidIt）**：支持双向遍历，在前向迭代器基础上增加`--`
5.  **随机访问迭代器（RanIt）**：支持任意位置随机访问，增加`[]`、`+`、`-`、`+=`、`-=`、关系比较

#### （2）相容规则
功能强的迭代器可以替代功能弱的迭代器：
`随机访问迭代器 → 双向迭代器 → 前向迭代器 → 输入/输出迭代器`

#### （3）各容器的迭代器类型
- 随机访问迭代器：vector、deque、basic_string
- 双向迭代器：list、map/multimap、set/multiset
- stack、queue 不支持迭代器遍历

### 3. 算法（Algorithm）
STL提供通用函数模板，通过迭代器操作容器元素，一套算法可适配多种容器。

#### （1）算法分类
调序算法、编辑算法、查找算法、算术算法、集合算法、堆算法、元素遍历算法

#### （2）使用特点
- 算法接收**迭代器区间**而非容器本身，因此与容器解耦
- 不同算法对迭代器类型有最低要求（如`sort`要求随机访问迭代器）

#### （3）自定义操作（谓词/函数对象）
算法支持传入自定义函数或函数对象，定制比较规则、操作逻辑。

##### 示例：自定义排序规则
```cpp
sort(v.begin(), v.end());  // 默认按<升序

// 自定义降序比较函数
bool greater(int x1, int x2) {return x1 > x2;}
sort(v.begin(), v.end(), greater);
```

##### 函数对象（仿函数）
重载了`()`运算符的类对象，可携带状态，适合灵活的自定义逻辑。
```cpp
class MatchMajor {
    Major major;
public:
    MatchMajor (Major m) { major = m; }
    bool operator() (Student& st) {
        return st.get_major() == major;
    }
};

// 统计软件工程专业人数
count_if(students.begin(), students.end(), MatchMajor(SOFTWARE));
```

#### （4）常用算法举例
- `max_element(first, last)`：返回区间最大值的迭代器
- `accumulate(first, last, 初始值, 操作)`：对区间元素累积运算
- `count_if(first, last, 条件)`：统计满足条件的元素个数
- `sort(first, last, 比较规则)`：对区间元素排序
