# 第6章 数据抽象：对象与类 复习笔记
## 一、面向对象程序设计基础
### 1. 核心思想
- 程序由若干**对象**组成，每个对象包含**数据**和**对数据的操作**。
- 操作数据通过向对象发送消息（调用成员函数）实现，不能直接操作对象内部数据。
- 对象的特征由**类**描述，类是对象的类型，对象是类的实例。
- 类可以通过**继承**复用其他类的特征。

### 2. 面向对象三大核心特性
| 特性 | 说明 |
|------|------|
| **封装** | 将数据与操作绑定为整体，通过访问控制隐藏内部实现，仅对外暴露接口 |
| **继承** | 子类复用父类的属性与方法，实现代码复用，支持单继承、多继承 |
| **多态** | 同一名字/消息具有多种解释，包括：函数重载、运算符重载、对象类型多态、消息多态 |

### 3. 绑定：多态的实现机制
- **静态绑定（前期绑定）**：编译阶段确定多态的具体实现，如函数重载、运算符重载。
- **动态绑定（后期绑定）**：运行阶段确定多态的具体实现，如继承中的虚函数重写。

### 4. 过程式 vs 面向对象（以栈为例）
| 对比维度 | 过程式程序 | 面向对象程序 |
|----------|------------|--------------|
| 数据与操作关系 | 数据和函数相互独立，数据作为函数参数传入 | 数据与操作封装为整体，操作是类的一部分 |
| 数据可见性 | 数据公开，可直接操作也可通过函数操作 | 数据私有，只能通过类提供的成员函数操作 |
| 初始化方式 | 需要手动显式调用初始化函数 | 创建对象时自动调用构造函数完成初始化 |

## 二、类的定义与成员语法
### 1. 类的定义格式
```cpp
class 类名 {
    成员描述（数据成员 + 成员函数）
};
```
- 类是用户自定义类型，属于**类型范畴**；对象是类的实例，属于**值范畴**。
- <span style="color:red">注意分号</span>

### 2. 数据成员语法规则
- 声明格式与普通变量一致，但**类内声明时禁止初始化**❗（静态数据成员除外）。
  ```cpp
  class A {
      int x = 0;       // 错误：类内不能初始化普通数据成员
      const double y = 0.0; // 错误：类内不能初始化const数据成员
  };
  ```
- 数据成员类型可以是任意C++类型（`void` 除外）。
- 若成员类型是未完整定义的类，只能声明为**指针或引用类型**，不能声明为对象。
  ```cpp
  class A; // 前向声明
  class B {
      A a;     // 错误：未见A的完整定义
      B b;     // 错误：B还未定义完 
      A *p;    // 正确：指针类型
      A &ref;  // 正确：引用类型
  };
  ```

### 3. 成员函数的两种定义方式
1.  **类内定义**：编译器默认按**内联函数**处理，适合短小函数。
    ```cpp
    class A {
        void f() { /* 函数体 */ }
    };
    ```
2.  **类外定义**：必须加 `类名::` 限定，声明在类内，实现在类外。
    ```cpp
    class A {
        void f(); // 声明
    };
    void A::f() { /* 函数体 */ } // 类外实现
    ```

### 4. 成员函数重载
- 除析构函数外，成员函数支持重载，遵循普通函数重载规则（参数个数/类型/顺序不同）。

### 5. 访问控制修饰符
C++ 通过三个关键字控制类成员的访问权限：
| 修饰符 | 访问范围 |
|--------|----------|
| `public` | 不受限制，类内、类外均可访问，构成类的对外接口 |
| `private` | 仅本类内部和友元可访问，类外不可直接访问 |
| `protected` | 本类、派生类、友元可访问，用于继承场景 |

#### 关键语法规则
- 类的**默认访问权限是 `private`**，未加修饰符的成员默认为私有。
- 一个类中可以出现多个 `public`/`private`/`protected` 块，作用域从当前修饰符到下一个修饰符或类结束。
- 设计原则：数据成员和内部工具函数设为 `private`，仅对外提供的操作设为 `public`。


## 三、对象的创建与操作
### 1. 对象的两种创建方式
#### （1）直接方式（栈/全局对象）
- 定义类类型的变量，通过**对象名**访问，分为全局对象和局部对象。
  ```cpp
  A a1;        // 创建单个对象
  A a2[100];   // 创建对象数组
  ```

#### （2）间接方式（动态堆对象）
- 运行时通过 `new` 分配内存，通过**指针**访问，用 `delete` 释放。
  ```cpp
  A *p = new A;      // 创建单个动态对象
  A *p = new A[100]; // 创建动态对象数组
  delete p;          // 释放单个对象
  delete []p;        // 释放对象数组，[]不可省略
  ```

#### 关键注意点
- `malloc`/`free` 不会调用构造函数和析构函数；`new`/`delete` 会自动调用。
- `new` 创建对象数组时，只能调用**默认构造函数**初始化。
- 释放对象数组时省略 `[]`，只会调用第一个对象的析构函数，会导致内存泄漏。

### 2. 对象的基本操作
- **成员访问**：对象用 `.` 运算符，对象指针用 `->` 运算符。
- **对象赋值**：同类对象可直接赋值，对应数据成员逐一拷贝。
- **取地址**：`&对象名` 可获取对象地址，赋值给对应类的指针。
- **函数传参/返回值**：对象可作为函数参数、返回值，值传递时会调用拷贝构造函数。

### 3. this 指针
#### 核心特性
- 每个非静态成员函数都有一个隐藏的形参 `this`，类型为 `类名 * const this`（指向当前对象的常指针）。
- 成员函数只有一份代码，通过 `this` 指针区分是哪个对象调用的函数。
- 编译时，`a.g(1)` 会被处理为 `A::g(&a, 1)`。

#### 使用场景
- 通常无需显式使用，编译器自动补全访问成员的 `this->`。
- 当需要把当前对象作为整体操作时（如返回当前对象、传递对象地址），需显式使用 `this`。


## 四、对象的初始化：构造函数
### 1. 构造函数基本特性
- 函数名与类名完全相同，**无返回值类型**
- 创建对象时**自动调用**，无需手动调用，用于初始化对象的数据成员。
- 支持重载，可提供多种初始化方式。

### 2. 默认构造函数
- 无参数，或所有参数都有默认值的构造函数。
- 若类中未定义任何构造函数，编译器会自动生成一个**隐式默认构造函数**（空实现）。
- 创建对象时未指定参数，自动调用默认构造函数。
```cpp
class A{ 
    ......
public:
    A();
    A(int i);
    A(char *p);
};
A a1; //调用默认构造函数。可写成A a1=A();
// 但不能写成A a1(); 会被解析为函数声明，不是创建对象
A a2(1); //调用A(int i)。可写成A a2=A(1); 或 A a2=1; 
A a3("abcd"); //调A(char *)。可写成A a3=A("abcd"); 或 A a3="abcd"; 
A a[4]; //调用对象a[0]、a[1]、a[2]、a[3]的默认构造函数。
A b[5]={A(), A(1), A("abcd"), 2, "xyz"}; //调用b[0]的A()、
//b[1]的A(int)、b[2]的A(char *)、b[3]的A(int)和b[4]的A(char *)
A *p1=new A; //调用默认构造函数。
A *p2=new A(2); //调用A(int i)。
A *p3=new A("xyz"); //调用A(char *)。
A *p4=new A[20]; //创建动态对象数组时调用各对象的默认构造函数
```

### 3. 成员初始化表🥦
#### 适用场景
- **初始化 `const` 数据成员、引用数据成员**（二者无法通过赋值初始化）。
- 初始化成员对象（指定调用成员对象的非默认构造函数）。

```cpp
class A {
    int x;
    const int y;
    int &z;
public:
    A(): y(1), z(x) { // 成员初始化表
        x = 0;
    }
};
```

#### 关键规则
- **初始化顺序由成员在类内的声明顺序决定，与初始化表的书写顺序无关**。
- 构造函数执行逻辑：先执行成员初始化表，再执行构造函数体。

### 4. 成员对象的初始化
类的数据成员可以是另一个类的对象（成员对象）。
- 若未在初始化表中指定，自动调用成员对象的**默认构造函数**。
- 可在初始化表中显式指定调用成员对象的某个构造函数。
- 构造函数执行顺序：**先成员对象->再自身**；
  多个成员对象按类内声明顺序构造。
```cpp
class A{
    int x;
public:
    A() { x = 0; }
    A(int i) { x = i; }
};
class B{ 
    A a;
    int y;
public:
    B(int i) { y = i;} //调用A的默认构造函数对a初始化。
    B(int i, int j): a(j) { y = i; } //调用A(int)对a初始化。
};
```

## 五、拷贝构造函数
### 1. 定义与格式
- 特殊的构造函数，参数是**本类对象的常引用**，用于用一个已有对象初始化新对象。
  ```cpp
  class A {
  public:
      A(const A &a); // 拷贝构造函数
  };
  ```

### 2. 自动调用拷贝构造的三种场景
1.  **定义对象时，用同类对象初始化**
    ```cpp
    A a1;
    A a2(a1);  // 等价于 A a2 = a1;
    ```
2.  **对象作为值参数传递给函数**：创建形参对象时，用实参拷贝初始化。
3.  **函数返回值为对象**：创建临时返回对象时，用函数内的局部对象拷贝初始化。

### 3. 隐式拷贝构造（浅拷贝）
- 若类中未定义拷贝构造函数，编译器自动生成隐式拷贝构造。
- 行为：**逐个成员拷贝**，普通成员直接赋值，成员对象调用其拷贝构造。
- 问题：当类包含指针成员时，浅拷贝会让两个对象的指针指向同一块内存，导致：
  1.  修改一个对象的指针内容会影响另一个；
  2.  对象消亡时同一块内存被释放两次，引发程序崩溃。

### 4. 自定义拷贝构造（深拷贝）
- 当类包含动态申请的资源（如指针指向的堆内存）时，必须自定义拷贝构造函数，实现深拷贝。
- 深拷贝：为新对象重新分配独立的资源，复制资源内容，而非只拷贝指针地址。
  ```cpp
  A::A(const A& a) {
      x = a.x;
      y = a.y;
      p = new char[strlen(a.p) + 1]; // 重新分配内存
      strcpy(p, a.p);                 // 复制内容
  }
  ```

### 5. 成员对象的拷贝处理
- 自定义拷贝构造时，不会自动调用成员对象的拷贝构造，需在初始化表中显式指定。
  ```cpp
  B::B(const B& b): a(b.a) { // 调用成员对象a的拷贝构造
      z = b.z;
  }
  ```

## 六、析构函数
### 1. 基本特性
- 函数名是 `~类名`，无返回值，**无参数**，因此不能重载。
- 对象消亡时**自动调用**，用于对象销毁前的清理工作（如释放动态申请的内存、关闭文件等）。

### 2. 调用时机
- 局部对象：函数执行结束、对象出作用域时调用。
- 动态对象：执行 `delete` 时调用。
- 全局对象：程序结束时调用。

### 3. 执行顺序
- 析构函数的执行顺序与构造函数**完全相反**：先执行自身析构体，再执行成员对象的析构。
- 若未自定义析构函数，编译器会生成隐式析构函数（空实现）。


## 七、const 成员
### 1. const 成员函数
#### 语法格式
- 声明和定义时，在函数参数列表后加 `const` 关键字。(放在类外定义时声明和定义都要加`const`)
  ```cpp
  // 类内声明
  void f() const;
  // 类外定义
  void A::f() const { /* 函数体 */ }
  ```

#### 核心作用
1.  **限制函数内部修改数据成员**：const 成员函数内不能修改任何普通数据成员的值，也不能调用非 const 成员函数。
    - 注意：仅修改指针指向的内容、不修改指针本身的值，编译器允许。
2.  **限制常量对象的操作**：`const` 修饰的对象（常量对象）只能调用 const 成员函数，不能调用非 const 成员函数。

## 八、静态成员
静态成员用于实现**同类所有对象共享数据/功能**，替代全局变量，符合封装原则。

### 1. 静态数据成员
- 所有对象共享同一份拷贝，不属于单个对象，存储在全局数据区。
- **类内声明，类外定义并初始化**，类外定义时不加 `static`，但要加类名限定。
  ```cpp
  class A {
      static int shared; // 类内声明
  };
  int A::shared = 0;    // 类外定义+初始化
  ```

### 2. 静态成员函数
- 没有隐藏的 `this` 指针，**只能访问静态成员**，不能直接访问非静态数据成员/成员函数。
- 属于类，不属于某个具体对象。

### 3. 静态成员的访问方式
- 对象访问：`a.get_shared()`
- 类名受限访问：`A::get_shared()`（推荐，更能体现静态属性）

### 4. 典型应用：对象计数
在类中定义静态计数变量，构造函数中计数+1，析构函数中计数-1，静态函数返回当前对象总数。

## 九、友元
### 1. 作用
在不破坏封装原则的前提下，让与类密切相关、又不适合作为类成员的程序实体，直接访问类的 `private`/`protected` 成员，提升访问效率。

### 2. 三种友元形式
1.  **友元全局函数**
    ```cpp
    class A {
        friend void func(); // 全局函数func是A的友元
    };
    ```
    
2.  **友元类**：B 类的所有成员函数都可以访问 A 类的私有成员。
    ```cpp
    class A {
        friend class B;//类 B 是类 A 的友元类
    };
    ```
3.  **友元类成员函数**：仅 C 类的 f 函数是 A 的友元。
    ```cpp
    class A {
        friend void C::f();//类 C 的成员函数C::f()是类 A 的友元函数
    };
    ```

### 3. 友元的性质
- **不对称性**：B 是 A 的友元，不代表 A 是 B 的友元。
- **不可传递性**：B 是 A 的友元、C 是 B 的友元，不代表 C 是 A 的友元。
- 友元是数据保护和访问效率的折中方案，仅对关系密切的实体使用。
