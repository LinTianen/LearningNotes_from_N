# 二叉搜索树

**二叉搜索树（Binary Search Tree，BST）** 也叫二叉查找树，是一种基于二叉树结构的有序数据结构，其核心特点是**便于高效的查找、插入和删除操作**，时间复杂度在理想情况下可达到 $O(\log n)$。

## 一、核心定义与性质
一棵二叉树要被称为二叉搜索树，需要满足以下**关键性质**：
1.  对于树中的任意一个节点 $x$：
    - 其**左子树**中的所有节点的值，都**小于** $x$ 的值；
    - 其**右子树**中的所有节点的值，都**大于** $x$ 的值。
2.  左右子树本身也必须是二叉搜索树（递归性质）。
3.  **唯一性**（可选约束）：大多数场景下要求树中没有重复值；若允许重复，通常会约定重复值存于左子树或右子树。

### 示例结构
```
        8
       / \
      3   10
     / \    \
    1   6    14
       / \   /
      4   7 13
```
这棵树满足二叉搜索树的性质：
- 根节点8的左子树所有节点（3、1、6、4、7）都小于8，右子树所有节点（10、14、13）都大于8；
- 节点3的左子树节点1小于3，右子树节点6大于3，以此类推。

## 二、基本操作
二叉搜索树的核心操作都是基于其有序性实现的，操作逻辑以递归或迭代的方式完成。

### 1. 查找（Search）
**目标**：查找值为 $target$ 的节点。
**思路**：
1.  从根节点开始比较；
2.  如果 $target =$ 当前节点值，查找成功，返回该节点；
3.  如果 $target <$ 当前节点值，进入左子树继续查找；
4.  如果 $target >$ 当前节点值，进入右子树继续查找；
5.  如果遍历到空节点仍未找到，说明树中无该值。

**时间复杂度**：理想情况 $O(\log n)$，最坏情况（退化为链表）$O(n)$。

```cpp
struct TreeNode {
    int val;
    TreeNode *left;
    TreeNode *right;
};

bool search(TreeNode* node, int target) {
    if (!node) return false;
    if (target == node->val) return true;

    if (target < node->val)
        return search(node->left, target); // 小往左
    else
        return search(node->right, target);// 大往右
}
```

### 2. 插入（Insert）
**目标**：插入值为 $val$ 的新节点，保持BST性质。
**插入的三种剧本**:
1.比当前节点小
这事儿我不归我管，归我左边的下属（左子树）管
2.比当前节点大
这事儿我不归我管，归我右边的下属（右子树）管
3.空树 (Base Case)
我就直接变成那个新节点

```cpp
TreeNode* insert(TreeNode* node, int val) {
    // 1. 递归出口: 找到空位
    if (node == NULL) {
        return createNode(val);
    }

    // 2. 递归查找并连接
    if (val < node->val) {
        node->left = insert(node->left, val);
    } else if (val > node->val) {
        node->right = insert(node->right, val);
    }

    // 3. 返回当前节点
    return node;
}
```

**注意**：插入操作不会破坏原有节点的相对顺序，新节点始终作为**叶子节点**插入。

### 3. 删除（Delete）
**目标**：删除值为 $val$ 的节点，保持BST性质。
**删除的三种情况**：
1.叶子节点 (Leaf)
无牵无挂，直接删除。
2.只有一个孩子
子承父业，独苗顶上 (父节点指向孙子)。
3.有两个孩子 (麻烦)
找替罪羊 (Successor) 顶包：
- 右子树中最小的节点 (或左最大)。
- 值覆盖当前节点，然后删除那个替罪羊。
```cpp
TreeNode* deleteNode(TreeNode* node, int key) {
    if (!node) return NULL;

    if (key < node->val)
        node->left = deleteNode(node->left, key);
    else if (key > node->val)
        node->right = deleteNode(node->right, key);
    else { // Found key
        // Case 1 & 2: 0或1个孩子
        if (!node->left) return node->right;
        if (!node->right) return node->left;

        // Case 3: 两个孩子
        TreeNode* min = node->right; // 找最小
        while(min->left) min = min->left; // 找最小
        node->val = min->val; // 1.值覆盖
        node->right = deleteNode(node->right, min->val);
    }
    return node;
}
```

## 三、遍历与有序性
二叉搜索树的一个重要特性是：**中序遍历的结果是一个严格递增的序列**。
以之前的示例树为例，中序遍历的顺序为：$1 \to 3 \to 4 \to 6 \to 7 \to 8 \to 10 \to 13 \to 14$，完全符合升序排列。

这一特性让二叉搜索树可以高效地完成**排序、范围查询**等任务。

## 四、优缺点
### 优点
1.  查找、插入、删除的平均时间复杂度为 $O(\log n)$，效率高于线性结构（如数组、链表）；
2.  中序遍历可直接得到有序序列，无需额外排序；
3.  结构灵活，支持动态插入和删除，优于有序数组（数组插入删除需移动元素）。

### 缺点
1.  **最坏情况退化为链表**：如果插入的元素是有序的（如依次插入1、2、3、4、5），二叉搜索树会退化成单链表，此时所有操作的时间复杂度变为 $O(n)$；
2.  树的性能高度依赖于插入顺序，不稳定。

## 五、改进与衍生结构
为了解决二叉搜索树退化成链表的问题，出现了**平衡二叉搜索树**，通过维持树的高度平衡，保证操作的最坏时间复杂度为 $O(\log n)$。常见的平衡二叉搜索树有：
1.  **AVL树**：严格平衡，任意节点的左右子树高度差不超过1；
2.  **红黑树**：近似平衡，通过颜色约束（红、黑节点）保证最长路径不超过最短路径的2倍，在工程中应用更广泛（如Java的 `TreeMap`、C++的 `map`）。

## 六、应用场景
1.  高效的动态查找、插入、删除场景（如字典、集合）；
2.  范围查询（如查找值在 $[a,b]$ 之间的所有节点）；
3.  作为平衡二叉树、B树、B+树的基础，衍生出数据库索引、文件系统等核心组件。
