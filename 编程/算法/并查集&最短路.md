# 算法笔记整理（重点加粗版）
## 一、并查集
并查集是一种维护若干元素组成的**不相交集合**的数据结构，核心支持两种操作：**合并两个集合**、**查询一个元素所属集合**（不支持集合分裂）。
其底层将集合表示为有根树，用树根代表集合，核心数组为 `fa[]`（记录每个节点的父亲）。

### 1. 初始化
每个元素独立成集合，树根的父亲为自身：
```cpp
for(int i=1;i<=n;++i)fa[i]=i;
// 扩展：C++库函数填充等差数列
iota(fa+1,fa+n+1,1);
```

### 2. 基础操作（未优化）
#### 查询
找到元素所属集合的根节点，时间复杂度 $O(n)$：
```cpp
int find(int x){return fa[x]==x?x:find(fa[x]);}
```

#### 合并
将两棵树合并（把一棵树的根挂到另一棵树下），含查询操作，时间复杂度 $O(n)$：
```cpp
// 后一种更优，x侧节点合并后深度更小
void merge(int x,int y){fa[find(x)]=find(y);}
```
注：需按需判断 $x,y$ 是否已在同一集合。

### 3. 核心优化（**重点**）
#### 优化1：查询 - 路径压缩
查询时将路径上所有节点直接挂到树根下，大幅降低后续查询复杂度：
```cpp
int find(int x){return fa[x]==x?x:fa[x]=find(fa[x]);}
```
仅路径压缩的并查集，单次操作平均复杂度 $O(\log n)$（实际场景表现优秀）。

#### 优化2-1：合并 - 启发式合并（**重要思想**）
合并时将**子树更小**的树挂到更大的树下，维护 `siz[]` 数组记录子树大小：
```cpp
void merge(int x,int y){
    if((x=find(x))==(y=find(y)))return; // 同一集合跳过
    if(siz[x]<siz[y])fa[x]=y,siz[y]+=siz[x];
    else fa[y]=x,siz[x]+=siz[y];
}
```
任意节点深度最大为 $\log n$，单次操作复杂度 $O(\log n)$，该思想可推广至其他数据结构的合并场景。

#### 优化2-2：合并 - 按秩合并
将启发式合并的 `siz[]` 替换为“秩” `rk[]`（表示树的最大深度，无需精确维护），合并时小秩挂到大秩下，复杂度同启发式合并（$O(\log n)$）。

### 4. 最终复杂度（**重点**）
路径压缩 + 启发式/按秩合并的并查集，单次操作复杂度 $O(\alpha(n))$（$\alpha$ 为阿克曼函数反函数，增长极慢，近似常数）。

### 5. 应用扩展
- 维护连通块
- 扩展域并查集（判断二分图、2-SAT）
- 带撤销并查集（**不可路径压缩**）

## 二、优先队列（堆）
堆用于维护元素的最大值/最小值，核心操作：**加入元素**、**查询最值**、**删除最值**，可直接使用C++ STL的`priority_queue`。

### 1. 基础定义与操作
| 类型       | 定义方式                                      | 核心操作                          |
|------------|-----------------------------------------------|-----------------------------------|
| 大根堆（默认） | `priority_queue<int> q;`                      | `q.push(x)`：加入元素<br>`q.top()`：查最大值<br>`q.pop()`：删最大值<br>`q.empty()`：判空<br>`q.size()`：查大小 |
| 小根堆     | `priority_queue<int,vector<int>,greater<int>> q;` | 同上，`top()` 返回最小值          |

大根堆与小根堆：https://www.doubao.com/thread/we64cebfb59fd109f
### 2. 结构体优先队列（**重点**）
需重载`<`运算符定义比较规则：
```cpp
struct node{
    int u,dis;
    // 重载方式1：成员函数
    bool operator <(const node &a)const{
        return dis<a.dis; // 按dis大根堆（返回true则当前节点优先级低）
    }
    // 构造函数（简化初始化）
    node(int a,int b):u(a),dis(b){}
};
// 重载方式2：全局函数
bool operator <(const node &a,const node &b){
    return a.dis<b.dis;
}
// 重载方式3：自定义比较结构体
struct cmp{
    bool operator ()(const node &a,const node &b){
        return a.dis<b.dis;
    }
};
priority_queue<node,vector<node>,cmp> q;
```

## 三、最短路
### 核心问题
给定图 $G=(V,E)$，边有权值 $w$，求两点间路径权值和最小的路径（单源/全源），需关注**负边权**、**负环**（负环会导致最短路不存在）。
核心性质：$dis(u)+w(u,v)\geq dis(v)$（松弛操作依据）。

### 1. Floyd（弗洛伊德）
**全源最短路**算法，基于动态规划，可处理负边权，**无法判断负环**。

#### 无穷大表示（**技巧**）
```cpp
const int INF1=0x3f3f3f3f; // int型，避免溢出
const long long INF3=0x3f3f3f3f3f3f3f3f; // long long型
// 快速赋值无穷大
memset(dis,0x3f,sizeof(dis));
```

#### 核心代码（**重点**）
```cpp
// k必须在最外层！
for(int k=1;k<=n;++k)
    for(int i=1;i<=n;++i)
        for(int j=1;j<=n;++j)
            dis[i][j]=min(dis[i][j],dis[i][k]+dis[k][j]);
```
本质：`f[k][i][j]` 表示仅经过1~k节点中转的 $dis(i,j)$，省略第一维后需保证k在外层。

### 2. SPFA
**单源最短路**算法，可处理负权、判断负环，基于Bellman-Ford的队列优化（仅更新过的点能松弛邻点）。

#### 核心操作：松弛
$dis(v)=min(dis(v),dis(u)+w(u,v))$（尝试用u的最短路更新v）。

#### 核心代码
```cpp
void SPFA(int s){
    memset(dis,0x3f,sizeof(dis));
    memset(vis,0,sizeof(vis)); // 标记是否在队列
    static int q[maxn];int l=1,r=0;
    dis[q[++r]=s]=0,vis[s]=1;
    while(l<=r){
        int u=q[l++];vis[u]=0;
        for(auto [v,w]:e[u])
            if(dis[u]+w<dis[v])
                dis[v]=dis[u]+w,!vis[v]&&(vis[q[++r]=v]=1);
    }
}
```

#### 关键说明
- 负环判断：若某点的最短路经过超过n个点，说明存在负环；
- 时间复杂度：最坏 $O(nm)$（易被卡），01最短路（边权仅0/1）可优化至 $O(n+m)$。

### 3. Dijkstra（迪杰斯特拉）
**单源最短路**算法，仅适用于**无负边权**场景，基于贪心+堆优化。

#### 核心思想
将点分为“黑点（已确定最短路）”和“白点（未确定）”，每次选白点中dis最小的点染黑，用其松弛邻点。

#### 堆优化核心代码（**重点**）
```cpp
struct node{
    int u,dis;
    node(int a,int b):u(a),dis(b){}
    bool operator <(const node &a)const{
        return dis>a.dis; // 小根堆（优先队列默认大根堆，反向定义）
    }
};
bool vis[maxn];
int dis[maxn];
inline void dijkstra(int s){
    memset(dis,0x3f,sizeof(dis));
    memset(vis,0,sizeof(vis));
    priority_queue<node> q;
    q.push(node(s,dis[s]=0));
    while(!q.empty()){
        int u=q.top().u;q.pop();
        if(vis[u])continue;vis[u]=1;
        for(int i=head[u],v;i;i=e[i].next)
            if(!vis[v=e[i].to]&&dis[u]+e[i].w<dis[v])
                q.push(node(v,dis[v]=dis[u]+e[i].w));
    }
}
```
#### 时间复杂度
堆优化后为 $O(m\log n)$，斐波那契堆可优化至 $O(n\log n+m)$（不常用）。

通俗理解：https://www.doubao.com/thread/w09885e80c66e3fff

## 四、最小生成树（MST）
### 基本概念
- 生成子图：点集不变，边集为原图子集；
- 生成树：生成子图为树（连通、无环、n-1条边）；
- 最小生成树：边权和最小的生成树（仅连通图存在）。

### 1. Kruskal（克鲁斯卡尔）
**前置知识：并查集**，核心思想：贪心。

#### 核心过程（**重点**）
1. 所有边按权值**从小到大排序**；
2. 初始所有点孤立，逐个遍历边：
   - 若边的两个端点不在同一连通块（并查集查询），则加入该边，合并两点；
   - 若已连通，跳过；
3. 直到加入n-1条边，形成生成树。

#### 正确性
归纳+反证：加入的边若不在最优解中，必形成环，且环上无更小权值边（否则已被优先加入），替换后仍为最优解。

#### 时间复杂度
瓶颈为边排序，$O(m\log m)$（$m$ 为边数），**实际更常用**。

### 2. Prim（普里姆）
核心思想：贪心，从单个点出发，逐步向树中加边（选当前树向外扩展的最小权值边），直到包含所有点。
时间复杂度：朴素版 $O(n^2)$，堆优化版 $O(m\log n)$。

### 3. 关键扩展（**重点**）
- 瓶颈生成树：最大边权最小的生成树，**等价于最小生成树**；
- Kruskal重构树（进阶，适用于路径最值查询等场景）；
- 次小生成树（https://oi-wiki.org/graph/mst/#%E6%AC%A1%E5%B0%8F%E7%94%9F%E6%88%90%E6%A0%91）。
