# KMP算法详细笔记
## 一、算法概述
KMP算法是一种高效的字符串匹配算法，全称为Knuth-Morris-Pratt算法。其核心优势在于避免了朴素字符串匹配中大量不必要的字符回溯，通过预处理模式串得到一个前缀函数（也称为next数组），利用该数组在匹配过程中跳过无效比较，从而将时间复杂度优化至O(n+m)（其中n为文本串长度，m为模式串长度）。

## 二、核心概念：前缀函数（next数组）
### 1. 定义
对于模式串`b`（长度为m），前缀函数`net[i]`（部分资料记为`next[i]`）表示模式串前`i`个字符组成的子串`b[1..i]`中，**最长的相等前缀和后缀的长度**。（子串不包括全串）
- 前缀：从字符串起始位置开始的子串（不包含最后一个字符）；
- 后缀：以字符串末尾位置结束的子串（不包含第一个字符）；
- 特殊规定：`net[1] = 0`（单个字符无有效前缀和后缀）。

### 2. 作用
在匹配过程中，若文本串`a`的第`i`个字符与模式串`b`的第`j+1`个字符不匹配，无需回溯`i`，只需将`j`更新为`net[j]`，再继续比较`a[i]`与`b[j+1]`，避免重复比较已匹配成功的前缀部分。

## 三、算法实现代码
### 1. 前缀函数（net数组）初始化（kmp_init函数）
```c
// m :模式串长度
// b :模式串

int net[N];  // N为预设的最大字符串长度
void kmp_init() {
    net[1] = 0;  // 初始化第一个字符的前缀函数值为0
    // i从2开始（遍历模式串前i个字符），j记录当前最长匹配前缀长度
    for (int i = 2, j = 0; i <= m; i++) {  
        // 若当前后缀与前缀不匹配，回溯j至net[j]，直到j=0或匹配成功
        while (j > 0 && b[j+1] != b[i]) j = net[j];

        // 若匹配成功，j向后移动一位（延长最长匹配长度）
        if (b[j+1] == b[i]) j++;
        net[i] = j;  // 记录前i个字符的最长相等前缀后缀长度
    }
}
```
- 输入：模式串`b`（下标从1开始）、模式串长度`m`；
- 输出：前缀函数数组`net`。

### 2. KMP匹配主函数（kmp函数）
```c
// m :模式串长度
// n :主串长度

int f[N];  // f[i]表示文本串前i个字符与模式串的最长匹配长度
void kmp() {
    for (int i = 1, j = 0; i <= n; i++) {  // i遍历文本串，j记录当前模式串匹配长度
        // 若不匹配且j>0，回溯j；同时避免j超过模式串长度（j==m时已匹配成功）
        while (j > 0 && (j == m || b[j+1] != a[i])) j = net[j];
        // 若当前字符匹配，j向后移动一位
        if (b[j+1] == a[i]) j++;
        f[i] = j;  // 记录文本串前i个字符的最长匹配长度
    }
}
```
- 输入：文本串`a`（下标从1开始）、文本串长度`n`、模式串`b`、前缀函数数组`net`；
- 输出：匹配状态数组`f`，若`f[i] == m`，说明文本串第`i-m+1`至`i`位与模式串完全匹配。

## 四、典型例题解析
### 例题1：子串查找（核心应用）
#### 问题描述
给定文本串`a`和模式串`b`，查找`b`在`a`中所有出现的起始位置。

#### 解题思路
1. 对模式串`b`调用`kmp_init()`生成`net`数组；
2. 调用`kmp()`函数进行匹配，遍历`f`数组；
3. 若`f[i] == m`，则起始位置为`i - m + 1`（下标从1开始）。

#### 示例（结合PPT隐含逻辑）
- 文本串`a = "ABCABCDABABCDABCDABDE"`
- 模式串`b = "ABCDABD"`
- 匹配结果：在`a`的第5位和第13位出现`b`。

### 例题2：重复子串
#### 问题描述
判断一个字符串`b`是否能由某个子串重复多次构成（如`"ababab"`可由`"ab"`重复3次构成）。

#### 解题思路
1. 计算字符串`b`的前缀函数`net`数组，获取`net[n]`（`n`为字符串长度）；
2. 若`n % (n - net[n]) == 0`，则字符串可由长度为`n - net[n]`的子串重复构成；否则不能。

#### 代码片段（PPT核心逻辑）
```c
n = strlen(b + 1);  // 假设字符串b下标从1开始，获取长度n
pre();  // 即kmp_init()，计算net数组
if (n % (n - net[n]) == 0) {
    // 可分解为重复子串，输出重复次数或子串（根据题目要求）
} else {
    puts("1");  // 不可分解，输出1（表示自身为最小单元）
}
```
#### 原理
- `n - net[n]`是字符串的最小周期长度：若前缀函数`net[n]`表示最长相等前缀后缀长度，则剩余部分`n - net[n]`是无法再缩短的最小重复单元；
- 若`n`能被该最小周期长度整除，则整个字符串由该单元重复构成。

### 例题3：周期长度和
#### 问题描述
给定字符串，计算其所有可能的周期长度之和（周期：若存在`d`使得对于所有`i`，`s[i] = s[i+d]`，则`d`为周期）。

#### 核心逻辑（结合PPT图示）
1. 周期的性质：字符串的周期`d`满足`n - net[n] ≤ d ≤ n`（`n`为字符串长度）；
2. 从`net[n]`开始回溯，每次取`net[j]`（`j`从`net[n]`逐步减小至0），对应的`n - j`均为有效周期；
3. 累加所有有效周期长度，得到结果。

#### 示例
- 字符串`"ababab"`，`n=6`，`net[6]=4`；
- 回溯过程：`j=4`→`n-j=2`（周期）；`j=net[4]=2`→`n-j=4`（周期）；`j=net[2]=0`→`n-j=6`（周期）；
- 周期长度和：`2+4+6=12`。

### 例题4：子串拆分
#### 问题描述
给定字符串`a`和整数`k`，拆分字符串`a`为若干个子串，要求每个子串的长度至少为`k`，且每个子串都是某个模式（通常为自身前缀）的匹配结果，统计合法拆分的数量。

#### 解题思路（PPT代码逻辑）
1. 预处理字符串`a`的前缀函数`net`数组（此处`a`既作为文本串也作为模式串）；
2. 遍历字符串，对于每个位置`i`，通过前缀函数回溯`j`，确保`j`对应的前缀长度满足`(j << 1) >= i`（即最长匹配前缀不超过当前子串长度的一半，避免重复拆分）；
3. 若`j >= k`，说明当前位置可作为一个拆分点，累加答案`ans`。

#### 代码片段
```c
for (int i = 2, j = 0; i <= n; i++) {
    // 计算前缀函数net[i]（此处省略net数组存储，直接用j维护）
    while (j > 0 && a[j+1] != a[i]) j = net[j];
    if (a[j+1] == a[i]) j++;
    // 确保匹配前缀不超过当前子串长度的一半，避免无效拆分
    while ((j << 1) >= i) j = net[j];
    if (j >= k) ans++;  // 满足长度要求，拆分计数+1
}
```

## 五、关键补充说明
1. 字符串下标约定：PPT中代码均采用“1-based”下标（字符串从第1位开始计数），实际实现时需注意与“0-based”下标的转换（核心逻辑一致，仅边界条件调整）；
2. 前缀函数的本质：是对模式串自身结构的分析，找到“前缀=后缀”的最长重叠部分，从而实现匹配时的高效回溯；
3. 算法适用场景：除上述例题外，KMP还可用于字符串去重、最长回文子串（结合其他技巧）、多模式匹配（扩展为AC自动机）等场景。
