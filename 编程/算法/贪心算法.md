# 贪心算法详细笔记
## 一、贪心算法基础概念
### 1. 核心思想
贪心算法基于**贪心思想**，核心是：每次做决策时，总是采取当前意义下的最优策略，通过“局部最优”推至“整体最优”。
- 正面示例：收银员找钱时，优先选择面值最大的纸币，以快速完成找零。
- 反面示例：带权树中求根节点到叶子节点的最大权值路径，若每步贪心选择权值较大的子节点，可能错过全局最优路径（需严谨证明贪心策略的正确性）。
- 通俗理解：贪心是“下意识的局部最优选择”，其结果可能正确也可能错误，需结合问题特性判断。

### 2. 贪心算法的适用条件
利用贪心法求解的问题必须具备以下两个核心特征：
#### （1）贪心选择性质
- 定义：问题的整体最优解可通过一系列局部最优解的选择实现，且每次选择仅依赖之前的决策，不依赖后续未做出的选择。
- 关键：需证明每一步的贪心选择最终能导向整体最优解（不可直接默认局部最优即全局最优）。

#### （2）最优子结构性质
- 定义：当问题的最优解包含其子问题的最优解时，称该问题具有最优子结构性质。
- 意义：是贪心算法适用的关键前提，确保局部最优解的累积能形成全局最优解（与动态规划的最优子结构性质本质一致，但决策方式不同）。

## 二、贪心策略的证明手段
贪心策略的正确性不能直观判断，需通过以下严谨方法证明：
### 1. 反证法
- 逻辑：假设“贪心策略不成立”（即存在更优的非贪心选择），推导得出矛盾结论，从而证明原贪心策略的正确性。
- 适用场景：简单贪心策略的验证（如找零问题、区间选点问题）。

### 2. 数学归纳法
- 逻辑：先证明基础情况（如n=1时）贪心策略成立，再假设n=k时成立，推导n=k+1时也成立，最终归纳出一般情况的正确性。
- 适用场景：具有递推结构的贪心问题（如排序类贪心、资源分配问题）。

### 3. 决策包容性
- 逻辑：证明在任意局面下，做出局部最优决策后，问题状态空间的可达集合包含了所有其他决策的可达集合。
- 核心：局部最优策略提供的可能性覆盖了其他所有策略，因此不会因当前选择错过全局最优解。

### 4. 范围缩放
- 逻辑：证明局部最优策略的作用范围扩展后（如从单个元素扩展到多个元素、从局部区间扩展到全局区间），整体结果不会变差。
- 适用场景：策略可逐步扩展的问题（如覆盖问题、分配问题）。

### 5. 微扰（邻项交换）法
- 逻辑：证明在任意局面下，对局部最优策略进行微小调整（如交换相邻元素的选择顺序），会导致整体结果变差。
- 典型应用：以“排序”为核心的贪心策略证明（如活动安排、任务调度问题中按结束时间排序的策略验证）。

## 三、贪心算法的配套工具与技能
### 1. 核心配套工具
- STL容器：`priority_queue`（优先队列，本质是堆结构，用于快速获取当前最优元素，如大根堆取最大值、小根堆取最小值）。
- 数据结构：Trie树（适用于字符串相关的贪心问题，如前缀匹配、字典序最优选择）。
- 基础操作：排序（贪心策略的核心支撑，如按区间右端点、任务优先级等排序）。

### 2. 必备技能：重载运算符
在C++中，使用`priority_queue`存储自定义结构体时，需重载`<`运算符以定义排序规则（即“最优”的判断标准）。

#### （1）示例代码
```cpp
// 结构体定义（存储学生姓名和分数）
struct student {
    string name;
    int score;
    // 重载<运算符：分数高的优先；分数相同则名字字典序小的优先
    bool operator<(const student a) const {
        return score < a.score || (score == a.score && name > a.name);
        // 完整表达式（隐含this指针）：
        // this->score < a.score || (this->score == a.score && this->name > a.name);
    }
};
// 定义优先队列（大根堆，按重载后的规则排序）
priority_queue<student> pq;
```

#### （2）关键语法解释：`const student& a`
- `&`（引用）的作用：
  1. 避免拷贝开销：传递引用而非对象副本，对于包含字符串、数组等成员的大型结构体，可节省拷贝构造和析构的性能消耗。
  2. 直接操作原始对象：引用是原对象的别名，对引用的操作等同于操作原对象（但因`const`限制，实际不可修改）。
- `const`（常量）的作用：
  1. 保护原数据完整性：禁止函数内部通过引用修改原对象成员（如`a.score = 100`会触发编译错误），避免意外副作用。
  2. 扩展适用范围：`const`引用可绑定到临时对象（如`student("Tom", 90)`）或常量对象（如`const student s`），提升函数通用性。

### 3. 贪心的拓展：反悔贪心
- 核心思想：先按贪心策略做出局部最优选择，后续若发现该选择导致全局结果变差，可“撤销”该选择，替换为更优的选择。
- 适用场景：单次贪心无法覆盖所有最优可能性的问题（如资源分配、任务调度中的动态调整场景）。

## 四、贪心算法经典例题
### 例题1：奶牛晒衣服（堆的应用）
#### 1. 问题背景
有n件衣服，每件衣服初始湿度为`h[i]`，每轮晾晒后，用烘干机处理的衣服湿度会减少B（未处理的衣服湿度自然减少A，本题代码中未体现自然减少，核心逻辑为优先处理湿度最大的衣服），求所有衣服湿度降至0以下所需的最少轮数。

#### 2. 贪心策略
每次选择当前湿度最大的衣服进行烘干（局部最优：优先处理最难干的衣服，以最快降低整体最大湿度）。

#### 3. 代码实现
```cpp
#include <cstdio>
#include <queue>
using namespace std;

int main() {
    int n, A, B;
    scanf("%d%d%d", &n, &A, &B);
    int h[100010];
    priority_queue<int> q; // 大根堆，默认每次取出最大值
    for (int i = 1; i <= n; i++) {
        scanf("%d", &h[i]);
        q.push(h[i]); // 将所有衣服湿度入堆
    }
    int t = 0;
    while (!q.empty() && q.top() > 0) { // 存在未晒干的衣服
        t++; // 轮数加1
        int x = q.top(); // 取出当前湿度最大的衣服
        q.pop();
        q.push(x - B); // 烘干后湿度减少B，重新入堆
    }
    printf("%d\n", t);
    return 0;
}
```

#### 4. 代码解释
- 用`priority_queue`（大根堆）维护衣服湿度，确保每次能快速获取湿度最大的衣服。
- 每轮处理后，将烘干后的衣服湿度重新入堆，重复直至所有衣服湿度≤0。
- 时间复杂度：O(n log n)（堆的插入、删除操作均为O(log n)，共需O(n)轮）。

### 例题2：雷达装置（区间选点问题）
#### 1. 问题背景
在x轴上安装雷达，雷达探测范围为以自身为圆心、半径为d的圆。给定n个岛屿的坐标(x, y)，判断是否能通过安装雷达覆盖所有岛屿，若能，求最少需要的雷达数量。

#### 2. 核心转化
将每个岛屿转化为x轴上的区间：
- 若岛屿y坐标>d：雷达无法覆盖（直接返回-1）。
- 否则，雷达需安装在区间`[x - sqrt(d² - y²), x + sqrt(d² - y²)]`内，才能覆盖该岛屿。
- 问题转化为：在x轴上选择最少的点，使得每个区间都包含至少一个点（区间选点问题）。

#### 3. 贪心策略
1. 将所有区间按**右端点从小到大**排序。
2. 依次遍历每个区间：
   - 若当前区间已包含最后一个选择的雷达位置，直接跳过。
   - 若未包含，在当前区间的**右端点**安装新雷达（局部最优：选择区间右端点可最大程度覆盖后续区间，减少雷达数量）。

#### 4. 代码实现
```cpp
#include <cstdio>
#include <algorithm>
#include <cmath>
using namespace std;

const int N = 1010;
struct part { // 存储区间左右端点
    double l, r;
} a[N];
int x[N], y[N], n, d, ans;

// 重载<运算符，按区间右端点从小到大排序
bool operator<(part a, part b) {
    return a.r < b.r;
}

inline void work() {
    ans = 0;
    for (int i = 1; i <= n; i++) {
        cin >> x[i] >> y[i];
        if (d < y[i]) { // 岛屿超出雷达覆盖范围
            ans = -1;
            return;
        }
        double len = sqrt(1LL * d * d - 1LL * y[i] * y[i]);
        a[i].l = x[i] - len; // 区间左端点
        a[i].r = x[i] + len; // 区间右端点
    }
    sort(a + 1, a + 1 + n); // 排序区间
    ans = 1;
    double pos = a[1].r; // 第一个雷达安装在第一个区间的右端点
    for (int i = 2; i <= n; i++) {
        if (a[i].l <= pos && pos <= a[i].r) { // 当前区间已被覆盖
            continue;
        } else { // 未覆盖，新建雷达
            ans++;
            pos = a[i].r;
        }
    }
}

int main() {
    scanf("%d%d", &n, &d);
    work();
    printf("%d\n", ans);
    return 0;
}
```

#### 5. 证明（微扰法）
假设存在更优的选择（如在区间中间或左端点安装雷达），交换该点与区间右端点，会发现后续可覆盖的区间数量减少或不变，因此选择右端点是最优策略。

### 例题3：畜栏预定（区间分配问题）
#### 1. 问题背景
有n头牛，每头牛有固定的吃草时间区间`[l, r]`，每个畜栏同一时间只能容纳一头牛，求最少需要的畜栏数量，并输出每头牛对应的畜栏编号。

#### 2. 贪心策略
1. 将所有牛按**开始吃草时间（l）从小到大**排序。
2. 维护一个小根堆，存储每个畜栏中最后一头牛的结束时间和畜栏编号。
3. 依次遍历每头牛：
   - 若堆顶畜栏的最后一头牛结束时间≤当前牛的开始时间：复用该畜栏，更新堆中该畜栏的结束时间为当前牛的结束时间。
   - 若没有可复用的畜栏：新建畜栏，将其加入堆中。

#### 3. 代码实现
```cpp
#include <iostream>
#include <algorithm>
#include <queue>
using namespace std;

const int N = 50100;
int n;
struct cow { // 牛的吃草时间和原始编号
    int l, r;
    int id;
    // 重载<运算符，按开始时间从小到大排序
    bool operator<(const cow a) const {
        return l < a.l;
    }
} a[N];
int ans[N]; // 存储每头牛的畜栏编号

struct Node { // 小根堆元素：畜栏最后一头牛的结束时间、畜栏编号
    int r;
    int id;
    Node(int a, int b) : r(a), id(b) {}
    // 重载<运算符，小根堆（结束时间早的优先）
    bool operator<(const Node a) const {
        return a.r < r;
    }
};

int main() {
    cin >> n;
    for (int i = 1; i <= n; i++) {
        cin >> a[i].l >> a[i].r;
        a[i].id = i;
    }
    sort(a + 1, a + 1 + n); // 按开始时间排序
    priority_queue<Node> q;
    int cnt = 0; // 畜栏数量
    for (int i = 1; i <= n; i++) {
        if (q.empty() || q.top().r >= a[i].l) { // 无可用畜栏，新建
            cnt++;
            ans[a[i].id] = cnt;
            q.push(Node(a[i].r, cnt));
        } else { // 复用畜栏
            ans[a[i].id] = q.top().id;
            q.pop();
            q.push(Node(a[i].r, ans[a[i].id]));
        }
    }
    cout << q.size() << endl; // 畜栏总数（堆的大小）
    for (int i = 1; i <= n; i++) {
        cout << ans[i] << endl; // 每头牛的畜栏编号
    }
    return 0;
}
```

#### 4. 代码解释
- 排序：按开始时间排序确保先处理早开始的牛，避免资源浪费。
- 小根堆：快速获取当前最早空闲的畜栏（堆顶存储结束时间最早的畜栏）。
- 时间复杂度：O(n log n)（排序O(n log n)，堆操作O(n log n)）。

## 五、关键总结
1. 贪心算法的核心是“局部最优→全局最优”，但需满足贪心选择性质和最优子结构性质。
2. 贪心策略必须证明正确性，常用反证法、微扰法等。
3. 配套工具中，`priority_queue`（堆）和排序是贪心的核心支撑，重载运算符是自定义排序的关键。
4. 经典应用场景：区间问题（选点、覆盖、分配）、资源调度、找零问题等，核心思路是“排序+局部最优选择”。
