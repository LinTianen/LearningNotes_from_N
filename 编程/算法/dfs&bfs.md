# 图论基础与搜索算法笔记
## 一、图论基础
### （一）核心数据结构：vector
vector 是 STL 提供的**内存连续、长度可变**的动态数组（列表），兼具数组的随机访问优势和链表的动态扩展特性，是图论及算法实现中常用的基础数据结构。

#### 1. 核心特性
- 内存连续：支持常数复杂度（O(1)）的随机访问。
- 长度可变：可动态添加/删除元素，无需预先指定固定长度。
- 操作效率：末尾插入/删除为 O(1)，中间插入/删除为 O(n)（需移动后续元素）。

#### 2. 常用操作
| 操作                | 语法                          | 功能描述                                  |
|---------------------|-------------------------------|-------------------------------------------|
| 创建 vector         | vector<int> a(len, val)       | 创建长度为 len 的 vector，所有元素初始值为 val |
| 随机访问            | a[x]                          | 访问下标为 x 的元素（x 范围：0 ~ a.size()-1） |
| 末尾插入元素        | a.push_back(x)                | 在 vector 尾部添加元素 x                  |
| 末尾删除元素        | a.pop_back()                  | 删除 vector 尾部的最后一个元素            |
| 获取长度            | a.size()                      | 返回 vector 中当前元素的个数              |
| 遍历元素（范围 for） | for (auto x : a) { ... }      | 遍历 vector 中所有元素，x 为当前元素值    |
| 遍历元素（下标 for） | for (int i=0; i<a.size(); i++){ int x=a[i]; } | 通过下标遍历，可修改元素值 |

#### 3. 补充说明
- auto 关键字：在范围 for 中自动推导元素类型，简化代码（如 vector<double> 无需显式写 double x）。
- 下标越界风险：访问 a[x] 时需确保 x 在 [0, a.size()-1] 内，否则会触发未定义行为（建议结合 a.size() 判断边界）。

### （二）图的存储方式
图的存储核心是记录“顶点”和“边”的关联关系，需根据查询、遍历需求选择合适的存储结构，常见方式有4种：

#### 1. 直接存边（边集数组）
- 原理：用数组（或多个数组）存储每条边的完整信息，适用于边数较少、无需频繁查询单顶点邻边的场景。
- 存储形式：
  - 无向无权图：用一个数组存储二元组 (u, v)，表示 u 和 v 之间有边。
  - 有向带权图：用一个数组存储三元组 (u, v, w)，或三个独立数组分别存起点 u、终点 v、边权 w。
- 复杂度分析：
  - 查询某条边是否存在：O(m)（需遍历所有边，m 为边数）。
  - 遍历一个点的所有出边：O(m)（需遍历所有边筛选起点为该点的边）。
  - 遍历整张图：O(nm)（n 为顶点数，需对每个顶点遍历所有边）。
  - 空间复杂度：O(m)（仅存储所有边，无冗余）。
- 适用场景：稀疏图、临时存储边、需按边批量处理的场景（如 Kruskal 算法）。

#### 2. 邻接矩阵
- 原理：用 n×n 的二维数组 adj 存储图，通过数组元素的值表示顶点间是否有边，适用于顶点数少、需频繁查询边是否存在的场景。
- 存储形式：
  - 无向无权图：adj[u][v] = 1 表示 u 与 v 有边，adj[u][v] = 0 表示无边；且 adj[u][v] = adj[v][u]。
  - 有向带权图：adj[u][v] = w 表示 u 到 v 有边权为 w 的边，adj[u][v] = ∞（或特定值）表示无边。
- 复杂度分析：
  - 查询某条边是否存在：O(1)（直接访问数组元素）。
  - 遍历一个点的所有出边：O(n)（需遍历该顶点对应的整行数组）。
  - 遍历整张图：O(n²)（遍历整个二维数组）。
  - 空间复杂度：O(n²)（顶点数多时有大量冗余）。
- 适用场景：稠密图（边数接近 n²）、顶点数 n ≤ 1000 的场景（避免空间溢出）。

#### 3. 邻接表
- 原理：用“数组 + 动态列表”存储图，数组下标对应顶点，每个数组元素是一个列表，存储该顶点的所有邻接顶点（或边权），是图论中最常用的存储方式。
- 存储形式：
  - 定义：vector<vector<int>> adj(n+1); （n 为顶点数，顶点编号从 1 到 n）。
  - 添加边：adj[u].push_back(v) 表示添加 u 到 v 的有向边；无向图需同时添加 adj[v].push_back(u)。
  - 带权图：可定义为 vector<vector<pair<int, int>>> adj(n+1); ，pair 的第一个元素是邻接顶点，第二个元素是边权。
- 复杂度分析：
  - 查询某条边是否存在：O(d(u))（d(u) 为 u 的出度，需遍历 adj[u] 列表）。
  - 遍历一个点的所有出边：O(d(u))（直接遍历对应列表）。
  - 遍历整张图：O(n+m)（每个顶点和边各访问一次）。
  - 空间复杂度：O(n+m)（无冗余，仅存储实际存在的边）。
- 适用场景：稀疏图、需频繁遍历顶点邻边的场景（如 DFS、BFS、Dijkstra 算法）。

#### 4. 链式前向星
- 原理：用数组模拟链表实现的邻接表，解决 vector 邻接表在极端场景下的效率问题（如频繁插入删除），核心是通过“头指针 + 边数组”记录边的关联关系。
- 核心数组定义：
  - to[]：存储每条边的终点。
  - nxt[]：存储每条边的后继边（即同一起点的下一条边的索引）。
  - head[]：存储每个顶点的第一条边的索引（初始值为 -1，表示无初始边）。
  - cnt：边的计数器（初始值为 -1，用于分配边的索引）。
- 核心操作代码：
  ```cpp
  // 初始化：head[u] = -1（所有顶点初始无边），cnt = -1
  void add(int u, int v) {  // 添加 u 到 v 的有向边
      nxt[++cnt] = head[u];  // 当前边的后继指向 u 之前的第一条边
      head[u] = cnt;         // u 的第一条边更新为当前边
      to[cnt] = v;           // 当前边的终点为 v
  }
  // 遍历 u 的所有出边
  for (int i = head[u]; ~i; i = nxt[i]) {  // ~i 等价于 i != -1
      int v = to[i];  // v 是 u 的邻接顶点
  }
  ```
- 带权扩展：可新增 w[] 数组存储边权，add 函数中添加 w[++cnt] = weight 即可。
- 复杂度分析：与邻接表一致（O(n+m) 遍历、O(m) 空间）。
- 适用场景：大规模稀疏图、对内存使用有严格要求的场景（数组比 vector 更节省内存）。

### （三）辅助数据结构：栈与队列
#### 1. 栈（stack）
- 核心特性：**先进后出（LIFO）**，仅允许在栈顶进行插入/删除操作。
- 常用操作：
  - stack<int> S; ：创建存储 int 类型的栈。
  - S.push(x); ：在栈顶插入元素 x。
  - S.pop(); ：删除栈顶元素（无返回值，需先通过 top() 获取值）。
  - S.top(); ：获取栈顶元素（不删除）。
  - S.empty(); ：判断栈是否为空（空返回 true）。
  - S.size(); ：返回栈中元素个数。
- 适用场景：深度优先搜索（DFS）、递归模拟、括号匹配等。

#### 2. 队列（queue）
- 核心特性：**先进先出（FIFO）**，仅允许在队尾插入、队首删除。
- 常用操作：
  - queue<int> Q; ：创建存储 int 类型的队列。
  - Q.push(x); ：在队尾插入元素 x。
  - Q.pop(); ：删除队首元素（无返回值，需先通过 front() 获取值）。
  - Q.front(); ：获取队首元素（不删除）。
  - Q.empty(); ：判断队列是否为空。
  - Q.size(); ：返回队列中元素个数。
- 适用场景：广度优先搜索（BFS）、任务调度、最短路径（无权图）等。

## 二、深度优先搜索（DFS）
### （一）核心定义
深度优先搜索（DFS，Depth First Search）是一种**优先探索深度**的图遍历/搜索算法，其核心思想是：从起点出发，沿着一条路径一直探索到无法继续（即无未访问的邻接顶点），再回溯到上一个节点，选择另一条未探索的路径，直到遍历所有可达节点。

- 类比理解：类似“走迷宫”，遇到岔路选一条走到头，走不通就回头换另一条岔路。
- 问题空间映射：将“问题状态”视为图的“顶点”，“状态转移”视为图的“边”，DFS 本质是在状态图上的深度优先遍历，形成“搜索树”（成功递归的状态和转移构成的树）。
```cpp
#include <bits/stdc++.h>
using namespace std;
int main(){
    int n,m;//n个点，m条边
    cin >> n >> m;
    vector < vector < int > > nxt(n+1);
    vector < int > vis(n+1,0);
    for(int i=0;i<m;++i){
        int u,v;
        cin >> u >> v;
        nxt[u].push_back(v);
    }
    auto dfs =[&](auto &&self,int u)->void{
        vis[u]=1; cout << u << ' ';
        for(auto v:nxt[u]){
            if(!vis[v]) self(self,v);
        }
    };
    dfs(dfs,1);
    return 0;
}
```
### （二）DFS 序列的性质
- 序列定义：DFS 过程中首次访问节点的顺序称为“DFS 序列”。
- 核心性质：
  1. 每个节点仅被访问一次（通过“访问标记”避免重复）。
  2. 搜索树中，子节点的访问区间完全包含在父节点的访问区间内（递归嵌套特性）。
  3. 回溯是 DFS 的关键：当当前路径无法继续时，回退到上一状态，尝试其他可能，确保不遗漏路径。

### （三）剪枝优化
剪枝是 DFS 提高效率的核心手段，通过“舍弃无效路径”减少搜索次数，需遵循三大原则，常用四大策略：

#### 1. 剪枝三大原则
- 正确性：绝对不能剪去包含正确解的路径（剪枝的前提）。
- 准确性：尽可能多地剪去无效路径（剪枝的核心目标）。
- 高效性：剪枝判断本身需快速（避免判断开销抵消优化效果）。

#### 2. 常用剪枝策略
| 策略名称       | 核心思想                                                                 | 应用场景举例                     |
|----------------|--------------------------------------------------------------------------|----------------------------------|
| 优化搜索顺序   | 调整状态分支的探索顺序，优先探索更可能找到解的分支，减少无效搜索         | 数独游戏中优先填可选数字少的格子 |
| 排除等效冗余   | 若不同分支到达的子树等效（最终结果一致），仅探索其中一条分支             | 组合问题中避免重复排列（如选 [1,2] 和 [2,1] 等效） |
| 可行性剪枝（上下界剪枝） | 检查当前状态是否满足解的必要条件，若不满足则回溯（如超出范围、违反规则） | 拔河比赛中某组重量已超过总重一半 |
| 最优性剪枝     | 最优化问题中，若当前代价已超过已知最优解，无需继续探索该分支             | 最短路径问题中当前路径长度≥已知最短路径 |
| 记忆化         | 记录每个状态的搜索结果，重复访问时直接返回（避免重复计算）               | 斐波那契数列递归、动态规划结合DFS |

### （四）二进制的辅助应用
DFS 中常用二进制表示“状态”，简化代码并提高效率：
- 状态表示：用二进制的每一位 0/1 表示某个元素是否被选择（如第 i 位为 1 表示选择第 i 个元素）。
- 位运算优化：
  - &（与）：判断某一位是否为 1（如 (state & (1<<<i)) != 0 表示第 i 位被选择）。
  - |（或）：标记某一位为 1（如 state |= (1<<<i) 表示选择第 i 个元素）。
  - ^（异或）：翻转某一位状态（如 state ^= (1<<<i) 表示切换第 i 个元素的选择状态）。
- 补码应用：-x 的二进制是 x 按位取反加 1，利用 x & (-x) 可快速获取 x 的最低位 1（如用于提取当前选择的元素）。

### （五）例题解析
#### 1. 拔河比赛
- 问题核心：将 n 个物体分成两组，使两组重量和尽可能接近。
- DFS 思路：
  - 状态定义：当前处理到第 i 个物体，第一组的重量和 sum1，第二组的重量和 sum2。
  - 分支选择：第 i 个物体放入第一组（sum1 += w[i]）或第二组（sum2 += w[i]）。
  - 剪枝策略：
    - 可行性剪枝：若 sum1 已超过总重的一半，无需继续（此时 sum2 必然更小，差距只会扩大）。
    - 优化搜索顺序：先处理重量大的物体，更早触发剪枝。
- 关键优化：总重固定为 total，sum2 = total - sum1，只需跟踪 sum1，减少状态变量。

#### 2. 数独游戏
- 问题核心：在 9×9 网格中填入 1-9，满足行、列、3×3 宫格内无重复数字。
- DFS 思路：
  - 状态定义：当前网格的填充状态（用数组 mapp 存储，-1 表示未填充）。
  - 分支选择：对未填充的格子，选择合法数字（行、列、宫格内无重复）填充。
  - 剪枝与优化：
    - 优化搜索顺序：优先选择“可选数字最少的格子”填充，减少分支数。
    - 状态压缩：用三个数组 a[x]、b[y]、c[g] 记录行 x、列 y、宫格 g 已使用的数字（二进制位表示，如 a[x] 的第 v 位为 1 表示行 x 已用数字 v）。
    - 高效判断合法数字：state = a[x] & b[y] & c[g]，state 中未置 1 的位对应的数字即为合法选项。

### （六）DFS 进阶方向
- 双向搜索：从起点和终点同时出发，相遇时停止，适用于搜索空间较大的场景（如两点间最短路径）。
- 迭代加深搜索（IDA*）：结合 DFS 和 BFS 的优点，限制搜索深度，逐步增加深度，避免深层无效搜索。
- 启发式搜索（A*）：引入估价函数，优先探索更可能接近目标的路径，提高搜索效率。
- Dancing Links：用于解决精确覆盖问题（如数独、N皇后的优化），高效处理稀疏矩阵的回溯。

## 三、广度优先搜索（BFS）
### （一）核心定义
广度优先搜索（BFS，Breadth First Search）是一种**优先探索广度**的图遍历/搜索算法，其核心思想是：从起点出发，先访问起点的所有邻接节点（距离为 1 的节点），再依次访问每个邻接节点的邻接节点（距离为 2 的节点），直到遍历所有可达节点。

- 类比理解：类似“水波扩散”，从起点向外层层扩展，确保先访问距离近的节点。
- 核心优势：天然适合求解**无权图的最短路径问题**（首次访问节点时的路径即为最短路径）。

### （二）核心原理与实现步骤
#### 1. 核心数据结构
- 队列（queue）：存储待访问的节点，保证“先进先出”，确保按距离顺序访问。
- 距离数组（d[]）：记录每个节点到起点的最短距离（初始值为 0 或 -1，-1 表示未访问）。
- 访问标记数组（vis[]）：标记节点是否已访问，避免重复入队（可与距离数组合并，d[v] = -1 表示未访问）。

#### 2. 通用实现框架（无权图最短路径）
```cpp
#include<bits/stdc++.h>
using namespace std;

int main(){
    int n,m;
    cin >> n >> m;
    vector < vector < int > > nxt(n+1);
    vector < int > vis(n+1,0);
    for(int i=0;i<m;++i){
        int u,v;
        cin >> u >> v;
        nxt[u].push_back(v);
    }
    queue < int > a;
    auto bfs=[&]()->void{
        a.push(1);
        vis[1]=1;
        while(a.size()){
            int u=a.front();
            cout << u << ' ';
            a.pop();
            for(auto v:nxt[u]){
                if(!vis[v]){
                    vis[v]=1;
                    a.push(v);
                }
                
            }
        }
    };
    bfs();
    return 0;
}
```

### （三）例题解析：走迷宫
#### 1. 问题描述
在二维网格（迷宫）中，给定起点 (sx, sy) 和终点 (tx, ty)，网格中 '.' 表示可通行，'#' 表示障碍物，求从起点到终点的最短步数（上下左右四个方向移动）。

#### 2. BFS 实现思路
- 状态定义：每个节点是网格中的坐标 (x, y)，状态转移是向四个方向移动。
- 数据结构：
  - 队列：存储待访问的坐标，用 pair<int, int> 表示 (x, y)。
  - 距离数组：d[x][y] 表示 (x, y) 到起点的最短步数（初始为 -1）。
  - 方向数组：dx[] = {-1, 1, 0, 0}，dy[] = {0, 0, -1, 1}（四个方向的偏移量）。
- 核心步骤：
  1. 初始化：d[sx][sy] = 0，起点入队，标记为已访问。
  2. 队列非空时，取出队首坐标 (x, y)。
  3. 若 (x, y) 是终点，输出 d[x][y] 并返回。
  4. 遍历四个方向，计算新坐标 (nx, ny)。
  5. 检查 (nx, ny) 是否在网格内、是否可通行、是否未访问，满足则更新距离并入队。

#### 3. 完整代码实现
```cpp
#include <bits/stdc++.h>
using namespace std;
const int N = 100;  // 迷宫大小上限
char a[N][N];  // 迷宫网格
int d[N][N];  // 距离数组
bool vis[N][N];  // 访问标记（可与 d 合并）
int sx, sy, tx, ty;  // 起点、终点坐标
// 四个方向：上、下、左、右
int dx[] = {-1, 1, 0, 0};
int dy[] = {0, 0, -1, 1};

// 检查 (x, y) 是否在迷宫范围内
bool exist(int x, int y, int n, int m) {
    return x >= 0 && x < n && y >= 0 && y < m;
}

void bfs(int n, int m) {
    memset(d, -1, sizeof(d));
    memset(vis, false, sizeof(vis));
    queue<pair<int, int>> q;
    q.push({sx, sy});
    d[sx][sy] = 0;
    vis[sx][sy] = true;
    while (!q.empty()) {
        auto u = q.front();
        q.pop();
        int x = u.first, y = u.second;
        if (x == tx && y == ty) {
            printf("%d\n", d[x][y]);
            return;
        }
        // 遍历四个方向
        for (int i = 0; i < 4; i++) {
            int nx = x + dx[i];
            int ny = y + dy[i];
            // 检查合法性：在范围内、可通行（a[nx][ny] == '.'）、未访问
            if (exist(nx, ny, n, m) && a[nx][ny] == '.' && !vis[nx][ny]) {
                d[nx][ny] = d[x][y] + 1;
                vis[nx][ny] = true;
                q.push({nx, ny});
            }
        }
    }
    printf("-1\n");  // 无法到达终点
}
```

#### 4. 关键说明
- 二维坐标处理：用 pair<int, int> 存储坐标，或用一维数组映射（如 idx = x * m + y）。
- 边界检查：exist 函数确保坐标不超出迷宫范围（避免数组越界）。
- 可通行判断：根据题目条件调整（如例题中 '.' 可通行，若题目中 'e' 表示可通行，需修改为 a[nx][ny] == 'e'）。
- 无解处理：若队列空时未找到终点，输出 -1 表示无法到达。

### （三）BFS 的适用场景
- 无权图的最短路径问题（如迷宫、社交网络好友推荐）。
- 图的层次遍历（如二叉树的层序遍历）。
- 寻找两个节点之间的最短距离（如两点间最少步数）。
- 检测图中的环（如无向图中，若访问到已访问且非父节点的节点，则存在环）。

### （四）BFS 与 DFS 的核心区别
| 特性         | DFS                  | BFS                  |
|--------------|----------------------|----------------------|
| 探索顺序     | 深度优先（一条路走到头） | 广度优先（层层扩散）  |
| 数据结构     | 栈（递归本质是栈）    | 队列                 |
| 最短路径     | 不适合（可能找到长路径） | 适合（无权图首次访问即为最短） |
| 空间复杂度   | O(h)（h 为搜索深度）  | O(w)（w 为最大层节点数） |
| 适用场景     | 迷宫探索、排列组合、剪枝优化 | 最短路径、层次遍历、环检测 |
| 递归实现     | 容易（天然递归结构）  | 困难（需手动维护队列） |
