# C++ 继承与派生类 复习笔记

## 一、继承的基本概念
### 1. 软件复用与继承
- 软件复用：直接复用已有代码实现新功能，降低开发成本。
- 传统修改源码复用的缺点：源码难以获取、需要读懂源码、可靠性差易出错。
- **继承机制**：定义新类时，包含一个或多个已有类的全部功能，再补充新功能或重定义已有功能，是C++实现代码复用的核心机制。

### 2. 基类与派生类
- **基类（父类）**：被继承的已有类。
- **派生类（子类）**：通过继承生成的新类，拥有基类的所有特征，同时可定义新特征、重定义基类特征。
- 继承的分类：
  - **单继承**：一个类最多只有一个直接基类。
  - **多继承**：一个类可以有多个直接基类。

## 二、单继承
### 1. 定义语法
```cpp
class <派生类名> : [<继承方式>] <基类名>{
    <派生类新增成员说明表>
};
```
- 继承方式可选：`public` / `private` / `protected`，**默认继承方式为 private**。
- 示例：
```cpp
class A{  // 基类
    int x, y;
public:
    void f();
    void g();
};

class B : public A {  // 派生类，public方式继承A
    int z;       // 新增数据成员
public:
    void h();    // 新增成员函数
};
```

### 2. 派生类的成员构成
派生类的成员 = 从基类继承来的成员 + 自身新增的成员。
- **例外**：基类的**构造函数、析构函数、赋值运算符重载函数**不能被派生类继承。
- 示例：`B b;` 可调用 `b.f()`、`b.g()`（继承自A）、`b.h()`（自身新增）。

### 3. 派生类定义的前提与限制
1. **必须可见基类定义**：仅前置声明基类无法定义派生类，编译器需要知道基类的成员与内存大小。
2. **友元关系不可继承**：基类的友元不是派生类的友元；基类作为其他类的友元，派生类不自动成为该类的友元。
3. **派生类不能直接访问基类的私有成员**：只能通过基类提供的公有/保护成员函数间接访问。
```cpp
class A{
    int x, y;
public:
    void g() { /* 内部可访问x */ }
};
class B : public A{
public:
    void h()
    {
        // x = 1;  // 错误：x是基类私有成员，不可直接访问
        g();      // 正确：通过基类公有函数间接访问x
    }
};
```

### 4. 三种访问控制符
类成员的访问属性分为三类，控制不同场景的访问权限：

| 访问属性  | 类内部访问 | 类对象访问 | 派生类内部访问 |
|----------|-----------|-----------|---------------|
| public   | ✅ 可访问 | ✅ 可访问 | ✅ 可访问 |
| protected| ✅ 可访问 | ❌ 不可访问 | ✅ 可访问 |
| private  | ✅ 可访问 | ❌ 不可访问 | ❌ 不可直接访问 |

- `protected` 的核心作用：缓解封装与继承的矛盾，让基类成员对外部对象隐藏，但对派生类开放。

### 5. 三种继承方式与权限变化
继承方式决定了**基类成员在派生类中的访问属性**，规则如下表：

| 基类成员属性 \ 继承方式 | public 继承 | protected 继承 | private 继承 |
|------------------------|------------|---------------|-------------|
| public                 | public     | protected     | private     |
| protected              | protected  | protected     | private     |
| private                | 不可直接访问 | 不可直接访问 | 不可直接访问 |

- 记忆要点：
  - 基类 private 成员在任何继承方式下，派生类都不可直接访问。
  - public 继承：保持基类成员原有访问级别不变。
  - protected 继承：基类 public 成员降为 protected，其余不变。
  - private 继承：基类所有非 private 成员都降为 private。

### 6. 访问控制符的调整🥦
在任何继承方式下，除基类 private 成员外，可在派生类中显式调整基类成员的访问级别。
- 语法：
```
[public: | protected: | private:] 基类名::基类成员名;
```

- 示例：private 继承后，将部分基类成员调整为 public/protected
```cpp
class A{
public:
    void f1(); void f2(); void f3();
protected:
    void g1(); void g2(); void g3();
};
class B : private A{  // 私有继承，基类所有非私有成员默认变为private
public:
    A::f1;  // 将f1调整为public
protected:
    A::f2;  // 将f2调整为protected
    A::g1;
    A::g2;
};

B b;
b.f1();  // 合法，已调整为public
// b.f2(); // 错误，属于protected级别
```

### 7. 派生类中基类同名成员的处理
#### （1）同名隐藏规则
- 派生类定义了与基类**同名**的成员（函数/变量），会**隐藏**基类的同名成员；即使函数参数列表不同，也会隐藏基类所有同名函数。
- 若要访问被隐藏的基类成员，必须加**基类名作用域限定符** `基类名::成员名`。

示例：
```cpp
class A{
public:
    void f();
    void g();
};
class B : public A{
public:
    void f();       // 与基类同名，隐藏A::f
    void f(int);    // 参数不同，仍会隐藏A::f
    void h()
    {
        f();     // 调用B类自身的f
        A::f();  // 调用基类的f
        f(1);    // 调用B::f(int)
    }
};
B b;
b.f();     // 调用B::f
b.A::f();  // 调用A::f
```

#### （2）using 声明开放基类同名函数🥦
可以在派生类中用 `using 基类::函数名;` 将基类的同名函数引入派生类作用域，解除隐藏。
```cpp
class B : public A
{
public:
    using A::f;  // 把A类的f引入派生类作用域
    void f(int);
};
B b;
b.f();   // 等价于 b.A::f()，合法
b.f(1);  // 调用B::f(int)，合法
```

### 8. 派生类的构造与析构⭐
#### （1）构造函数调用规则
- 基类数据成员由基类构造函数初始化，派生类新增成员由派生类构造函数初始化。
- 创建派生类对象时，**先执行基类构造函数，再执行派生类构造函数**。
- 默认调用基类的默认构造函数；若要调用基类带参构造，必须在派生类构造函数的**初始化列表**中显式指定。

示例：
```cpp
class A{
    int x;
public:
    A() { x = 0; }
    A(int i) { x = i; }
};

class B : public A{
    int y;
public:
    B() { y = 0; }                  // 自动调用A()
    B(int i) { y = i; }             // 自动调用A()
    B(int i, int j) : A(i) { y = j; } // 显式调用A(int)
};

B b1;        // 执行顺序：A() → B()
B b2(1);     // 执行顺序：A() → B(int)
B b3(1,2);   // 执行顺序：A(int) → B(int,int)
```

#### （2）含成员对象时的构造/析构顺序
若派生类同时有基类和成员对象：
- **构造顺序**：基类构造 → 成员对象构造 → 派生类自身构造
- **析构顺序**：派生类析构 → 成员对象析构 → 基类析构（与构造相反）

#### （3）拷贝构造函数
- 编译器自动生成的隐式拷贝构造函数：会自动调用基类的拷贝构造函数。
- 自定义拷贝构造函数：默认调用基类的默认构造函数；如需调用基类拷贝构造，需在初始化列表显式指定。
```cpp
class D : public A
{
public:
    D() { ... }
    // 显式调用基类拷贝构造
    D(const D& d) : A(d) { ... }
};
```

### 9. 派生类的赋值运算符⭐
- 派生类**不继承**基类的赋值运算符。
- 编译器生成的隐式赋值运算符：自动调用基类赋值运算符，再逐个赋值派生类成员。
- 自定义赋值运算符：不会自动调用基类赋值，必须显式调用基类赋值。
```cpp
class B : public A
{
public:
    B& operator=(const B& b)
    {
        if (&b == this) return *this;  // 防止自赋值
        // 两种显式调用基类赋值的写法
        *(A*)this = b;                
        // 等价写法：this->A::operator=(b);
        // 再赋值派生类自身成员...
        return *this;
    }
};
```

## 三、代码复用的另一种方式：聚集（组合）
### 1. 概念
将一个类的对象作为另一个类的成员，通过调用成员对象的方法实现功能复用，也叫**组合**。
```cpp
class A { public: void f(); };
class B
{
    A a;  // A类对象作为成员
public:
    void f() { a.f(); }  // 复用A的f实现B的f
};
```

### 2. 继承 vs 聚集 对比
| 维度 | 继承 | 聚集（组合） |
|-----|-----|-------------|
| 适用场景 | "是一个"（is-a）的子类型关系 | "有一个"（has-a）的整体-部分关系 |
| 封装性 | 破坏封装，派生类可访问基类保护成员 | 不破坏封装，仅通过公有接口交互 |
| 子类型特性 | 支持，派生类对象可替代基类对象 | 不支持 |
| 耦合度 | 编译期确定，耦合度高 | 耦合度低，更灵活 |

## 四、虚函数与动态绑定⭐
### 1. 子类型与多态基础
public 继承下，派生类是基类的子类型，支持：
- 派生类对象可以赋值给基类对象（派生类独有成员被截断，称为对象切片）。
- 基类指针可以指向派生类对象。
- 基类引用可以引用派生类对象。

```cpp
A a; B b;
a = b;      // 合法，对象切片
A* p = &b;  // 合法，基类指针指向派生类对象
A& r = b;   // 合法，基类引用绑定派生类对象
```

### 2. 静态绑定（默认）
- **静态绑定**：编译阶段根据指针/引用的**静态类型**决定调用哪个类的函数，与实际指向的对象无关。
- C++ 默认采用静态绑定。
```cpp
void func(A* p) { p->f(); }  // 编译时确定调用A::f，与p指向的对象无关
```

### 3. 虚函数与动态绑定
- **动态绑定**：运行阶段根据指针/引用**实际指向的对象类型**决定调用的函数。
- C++ 通过**虚函数**实现动态绑定。

#### （1）虚函数定义语法
在基类的成员函数声明前加 `virtual` 关键字：
```cpp
class A{
public:
    virtual void f();  // 声明为虚函数
};
```

#### （2）虚函数的覆盖（重写 override）
派生类中定义与基类虚函数**同型构**的函数，构成对基类虚函数的覆盖，且自动成为虚函数（`virtual` 可省略）。

同型构要求：
1. 函数名完全相同
2. 参数列表（个数、类型、顺序）完全相同
3. 返回值类型相同，或返回基类返回类型的派生类（协变返回类型）

示例：
```cpp
class A
{
public:
    virtual void f();
    virtual A* clone();
};
class B : public A
{
public:
    void f();          // 覆盖A::f，自动为虚函数
    B* clone();        // 协变返回，合法覆盖
};
```

### 4. 虚函数的重要规则
1. 只有类的**成员函数**可以是虚函数，全局函数、静态函数不行。
2. **构造函数不能是虚函数**，析构函数可以且**通常建议声明为虚函数**（防止基类指针释放派生类对象时内存泄漏）。
3. 虚函数特性可继承：基类声明为虚，**派生类**同型构函数自动为虚，**`virtual` 可省略**。
4. 只有通过**指针或引用**调用虚函数时，才会触发动态绑定；对象直接调用不触发。
5. 在构造函数、析构函数中调用虚函数，**不进行动态绑定**，调用的是当前类自身的版本。🥦

### 5. 纯虚函数与抽象类
#### （1）纯虚函数
只有声明、没有实现的虚函数，语法：
```cpp
virtual 返回类型 函数名(参数列表) = 0;
```

#### （2）抽象类
- 包含**纯虚函数**的类称为抽象类。
- 抽象类**不能实例化对象**，仅用于作为基类被继承。
- 作用：为派生类提供统一的公共接口与框架，实现接口与实现分离。

示例：图形类抽象接口
```cpp
class Figure  // 抽象类
{
public:
    virtual void draw() const = 0;       // 纯虚函数
    virtual void input_data() = 0;       // 纯虚函数
};

class Rectangle : public Figure
{
public:
    void draw() const override { /* 具体实现 */ }
    void input_data() override { /* 具体实现 */ }
};
```

## 五、多继承
### 1. 定义语法
一个派生类同时继承多个直接基类：
```cpp
class <派生类名> : [<继承方式>] <基类名1>, [<继承方式>] <基类名2>, ...
{
    <派生类成员>
};
```
示例：
```cpp
class A { int m; public: void fa(); };
class B { int n; public: void fb(); };

class C : public A, public B
{
    int r;
public:
    void fc();
};
```

### 2. 构造与析构顺序
- 构造顺序：按**基类声明的先后顺序**依次调用基类构造函数，最后调用派生类构造。
- 析构顺序：与构造顺序完全相反。
- 示例中 `C c;` 构造顺序：`A()` → `B()` → `C()`。

### 3. 名字冲突与解决
多个基类拥有同名成员时，派生类直接调用会产生二义性。
- 解决方法：使用**基类名作用域限定**。
```cpp
class A { public: void f(); };
class B { public: void f(); };
class C : public A, public B
{
public:
    void func()
    {
        // f();  // 错误，二义性
        A::f();  // 正确，调用A的f
        B::f();  // 正确，调用B的f
    }
};

C c;
c.A::f();
```

### 4. 重复继承与虚基类
#### （1）重复继承（菱形继承）
派生类通过两条路径继承同一个间接基类，导致间接基类的成员在派生类中存在多份副本。
```cpp
class A { int x; };
class B : public A {};
class C : public A {};
class D : public B, public C {};
// D中包含两份x：B::x 和 C::x
```

#### （2）虚基类解决重复继承
将公共基类声明为**虚基类**，使派生类中只保留一份基类成员。
- 语法：继承时加 `virtual` 关键字🥦
```cpp
class B : virtual public A {};
class C : virtual public A {};
class D : public B, public C {};
// D中只有一份A的成员
```

#### （3）虚基类的构造规则
1. 虚基类的构造函数由**最派生类**（最终创建对象的类）的构造函数直接调用，中间类的构造中对虚基类的初始化会被忽略。
2. 虚基类构造函数**优先于**所有非虚基类构造函数执行。

示例：
```cpp
class A
{
    int x;
public:
    A(int i) { x = i; }
};
class B : virtual public A
{
    int y;
public:
    B(int i) : A(1) { y = i; }
};
class C : virtual public A
{
    int z;
public:
    C(int i) : A(2) { z = i; }
};
class D : public B, public C
{
    int m;
public:
    D(int i,int j,int k) : B(i), C(j), A(3) { m = k; }
};

D d(1,2,3);
// 构造顺序：A(3) → B(1) → C(2) → D(1,2,3)
// A的构造由最派生类D直接调用，x最终初始化为3
```
